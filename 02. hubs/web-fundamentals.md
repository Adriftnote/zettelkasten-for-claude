# 웹 기초 (Web Fundamentals)

웹은 **문서(HTML)**와 **스타일(CSS)**과 **동작(JS)**의 조합이다. 이들이 **HTTP 프로토콜**을 통해 브라우저와 서버 사이를 오간다.

---

## Observations

### 핵심 인사이트
- [insight] 웹의 본질은 **하이퍼링크로 연결된 문서**다. 나머지는 모두 그 위에 쌓인 레이어.
- [insight] HTML은 **구조**, CSS는 **표현**, JS는 **동작** - 관심사 분리의 전형
- [insight] [[Same-Origin Policy (동일 출처 정책)|SOP]]는 웹 보안의 근간. [[CORS (Cross-Origin Resource Sharing)|CORS]]는 그 예외를 허용하는 메커니즘
- [insight] Chrome Extension은 웹 페이지 위에서 동작하는 **별도의 실행 컨텍스트**

### 학습 경로
- [path] **입문**: [[HTML (HyperText Markup Language)|HTML]] → [[CSS (Cascading Style Sheets)|CSS]] → [[JavaScript (JS)|JS]] → [[DOM (Document Object Model)|DOM]]
- [path] **심화**: [[HTTP Methods]] → [[HTTP Status Codes]] → [[HTTP Header]] → [[Web Communication]]
- [path] **보안**: [[Same-Origin Policy (동일 출처 정책)|SOP]] → [[CORS (Cross-Origin Resource Sharing)|CORS]]
- [path] **확장**: [[Chrome Extension]] → [[Content Script]] → [[Background Script (Service Worker)|Background Script]]

### 인덱싱 (루만식)
- [index:1] [[HTML (HyperText Markup Language)]] - 웹 문서의 구조 정의
- [index:1a] [[DOM (Document Object Model)]] - HTML의 프로그래밍 인터페이스
- [index:1b] [[iframe (Inline Frame)]] - 문서 안의 문서
- [index:2] [[CSS (Cascading Style Sheets)]] - 시각적 표현과 스타일링
- [index:3] [[JavaScript (JS)]] - 동적 동작과 상호작용
- [index:4] [[HTTP Header]] - 웹 통신의 요청/응답 메타데이터
- [index:4a] [[HTTP Methods]] - GET, POST, PUT, DELETE 등 통신 방법
- [index:4b] [[HTTP Status Codes]] - 200, 404, 500 등 상태 응답
- [index:4c] [[Web Communication]] - 브라우저-서버 간 통신의 전체 그림
- [index:5] [[Same-Origin Policy (동일 출처 정책)]] - 웹 보안의 기본 정책
- [index:5a] [[CORS (Cross-Origin Resource Sharing)]] - 교차 출처 리소스 공유 메커니즘
- [index:6] [[Chrome Extension]] - 브라우저 기능 확장
- [index:6a] [[Content Script]] - 웹 페이지에 주입되는 스크립트
- [index:6b] [[Background Script (Service Worker)]] - 백그라운드 실행 환경
