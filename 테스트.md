# Claude Code & AI-Native Knowledge Management

## 1. 핵심 아키텍처: AI의 '눈'과 '손'
Claude(AI)는 단순히 텍스트를 생성하는 것이 아니라, 외부 도구를 사용해 시스템을 이해하고 조작함.

*   **MCP (Model Context Protocol):** AI가 외부 세계(내 컴퓨터, DB, API)와 소통하기 위한 **표준 플랫폼(OS)**.
*   **LSP (Language Server Protocol):** MCP 플랫폼 위에서 돌아가는 **코드 분석 전문 앱**. MS가 만든 오픈소스 표준으로, AI가 코드의 정의(Definition)와 참조(Reference)를 정밀하게 추적하게 함.
*   **결합 원리:** AI는 **자연어**로 전략을 짜고, **MCP**라는 통로를 통해 **LSP**라는 정밀 도구를 호출하여 필요한 정보만 핀셋으로 뽑아옴 (컨텍스트 효율화).

## 2. 시맨틱 인덱싱 (Semantic Indexing)
문서(.md)는 LSP가 없기 때문에 Claude Code는 내부적으로 별도의 인덱싱 과정을 거침.

*   **하이브리드 검색:** 키워드 기반의 **BM25**와 의미 기반의 **Vector 검색**을 결합.
*   **로컬 캐싱:** 내 컴퓨터(`~/.claude/` 등)에 **SQLite**나 **바이너리 DB** 형태로 프로젝트의 지도를 저장하여 재사용.
*   **Tree-sitter 활용:** 문서의 구조(헤더, 목록, 링크)를 파싱하여 단순 텍스트가 아닌 '의미적 구조체'로 파악.

## 3. 제텔카스텐(Zettelkasten) 최적화 전략
AI가 내 지식 저장소를 더 빠르고 정확하게 탐색하게 만드는 방법.

*   **폴더 네이밍:** 번호와 함께 '도메인 키워드' 명시 (예: `01. concepts-ai-coding`).
*   **인덱스 파일 활용:** 각 폴더에 `INDEX.md`를 두어 AI에게 해당 폴더의 '지도'를 선제적으로 제공.
*   **CLAUDE.md:** 프로젝트 루트에 전체 구조와 규칙을 정의하여 AI의 첫 단추를 교정.
*   **Metadata (YAML):** 노트 상단에 요약(Summary)과 태그를 작성하여 AI가 파일 전체를 읽지 않고도 내용을 파악하게 유도.

## 4. 관련 도구 및 프로젝트

*   **claw-code:** Claude Code의 로직을 Python/Rust로 재구현한 오픈소스 프로젝트. 내부 설계 연구에 용이함.
