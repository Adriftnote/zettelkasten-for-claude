## 1. 보고 체계 개요

### 1.1 계층 구조

```
개인(팀원/사무직 전원) → 팀(팀장) → 본부(본부장)
```

### 1.2 보고 주기 (4종)

| # | 보고서명 | 수행 시기 | 보고 내용 핵심 |
|---|---------|----------|---------------|
| 1 | **주간 실적 보고** | 매주 월요일 | 지난주 실적 + 이번주 계획 + 다짐 |
| 2 | **실적 점검 보고** | 매월 18~23일 중 첫 영업일 | 월 중간 KPI 진척율 점검 + 지원 요청 |
| 3 | **월간 실적 보고** | 매월 1~5일 중 첫 영업일 | 지난달 실적/원인분석/대책 + 이번달 계획 |
| 4 | **분기 실적 보고** | 익분기 첫달 1~5일 중 첫 영업일 | 지난분기 실적/원인분석/대책 (계획 없음) |

### 1.3 결재선

| 작성자 | 최종 결재권자 | 비고 |
|--------|-------------|------|
| 팀원 (전 보고) | 본부장 | |
| 팀장 (주간/실적점검) | 본부장 | |
| 팀장 (월간/분기) | **대표이사** | 월간부터 대표 결재 |
| 본부장 (전 보고) | **대표이사** | |

### 1.4 결재문서 제목 규칙

| 계층 | 형식 |
|------|------|
| 개인 | `{보고서명}의 건_20XX년 00월 00주차_00팀 홍길동` |
| 팀 | `{보고서명}의 건_20XX년 00월 00주차_00팀` (팀장 이름 없음) |
| 본부 | `{보고서명}의 건_20XX년 00월 00주차_00본부` |

- 주차 기준: 매달 첫번째 월요일 포함 주 = 1주차

### 1.5 수신자 및 후처리

- **수신자**: 경영기획팀 박세준 주임, 박승호 대리, 최은진 팀장
- **플로우 등록**: 결재문서번호 등록 및 원본파일 첨부 필수


## 7. 자동화 관점 핵심 포인트

### 7.1 데이터 흐름 (KPI cascading)

```
전사 KPI → 본부 KPI → 팀 KPI → 개인 KPI
           (본부장)    (팀장)    (팀원)
```

- 상위 KPI는 하위 보고서에서 참조 컬럼으로 표시
- 모든 KPI 행에 **실적검증여부** 필드 존재 (기본값: "검증대기")

### 7.2 공통 필드

| 필드 | 전 양식 공통 |
|------|-----------|
| 소속 | O |
| 직책/직급 | O |
| 사번 | O |
| 성명 | O |
| 주차/월/분기 | 보고 주기에 따라 |
| 기간 (시작~종료) | O |
| 작성일자 | O |

### 7.3 양식 간 재사용 가능 데이터

- **KPI 마스터**: 전사→본부→팀→개인 매핑 테이블 1회 정의 → 전 양식 참조
- **프로젝트 목록**: PM/담당 프로젝트 정보 → 주간/월간/분기/평가 모두 동일 구조
- **실적 데이터**: 주간 실적 누적 → 월간/분기 자동 집계 가능
- **분석 내용**: 월간 분석(원인/대책/일정) → 분기 보고에 재활용 가능
