## 용어 빠른 참조표

### 코드 관련

| 용어 | 한 줄 정의 | 파일 형태 |
|------|-----------|----------|
| 소스코드 | 사람이 작성한 프로그램 원본 | .py, .java, .c |
| 스크립트 | 즉시 실행 가능한 코드 | .sh, .js, .py |
| 바이너리 | 컴퓨터가 직접 실행하는 파일 | .exe, .app |
| 바이트코드 | 가상머신용 중간 코드 | .class, .pyc |

### 실행 도구

| 용어 | 역할 | 예시 |
|------|------|------|
| 컴파일러 | 소스 전체를 기계어로 변환 | gcc, clang |
| 인터프리터 | 코드를 한 줄씩 실행 | python, node |
| 어셈블러 | 어셈블리어를 기계어로 변환 | as, nasm |
| 링커 | 여러 파일을 하나로 합침 | ld |

### 코드 구조

| 용어 | 규모 | 비유 |
|------|------|------|
| 모듈 | 파일 하나 | 레고 블록 1개 |
| 패키지 | 여러 모듈 | 레고 세트 |
| 라이브러리 | 재사용 코드 모음 | 공구 세트 |
| 프레임워크 | 개발 골격 제공 | 조립식 가구 |

## 7. 학습 자료

### 추천 도서

- **입문**: "Hello Coding 한입에 쏙 파이썬" - 한빛미디어
  - 프로그래밍 완전 초보자용
  - Python으로 기초 개념 배우기
  - 일러스트 많아서 이해하기 쉬움

- **역사**: "CODE: 하드웨어와 소프트웨어에 숨어있는 언어" - 찰스 펫졸드
  - 컴퓨터와 프로그래밍의 역사
  - 원리를 깊이 있게 이해
  - 전자기기 팬들 필독서

- **컴퓨터 과학**: "컴퓨터 과학이 여는 세계" - 이광근
  - 서울대 컴퓨터공학부 교수 저술
  - 한국어 컴퓨터 과학 입문서 최고 평가
  - 알고리즘, 인공지능 개론도 포함

### 온라인 자료

- **생활코딩** (opentutorials.org)
  - 한국어 무료 강의
  - 초보자 친화적
  - 실전 코딩 기초

- **Khan Academy**
  - 컴퓨터 과학 기초 (영어)
  - 알고리즘 강의 최고
  - 자막 지원

- **CS50 (Harvard)**
  - 세계 최고의 CS 입문 강의 (무료)
  - 강의 영상 모두 유튜브에
  - 한국어 자막 지원

### 체험 사이트

- **Scratch** (scratch.mit.edu)
  - 블록 코딩 (어린이용)
  - 프로그래밍 개념 학습 최고
  - 게임/애니메이션 만들기

- **Replit** (replit.com)
  - 브라우저에서 코딩
  - 설치 없이 즉시 시작
  - 다양한 언어 지원

- **Codecademy**
  - 대화형 학습
  - 초보자에서 중급까지
  - 유료지만 가치 있음

## 8. 자주 묻는 질문 (FAQ)

### Q1: 프로그래밍 언어는 몇 개나 있나요?

**A**: 약 700개 이상! 하지만 실제로 많이 쓰이는 건 20~30개 정도입니다.

**Top 10 인기 언어** (2024년 기준):
1. Python (AI, 데이터 과학)
2. JavaScript (웹 개발)
3. Java (기업 시스템)
4. C++ (게임, 성능)
5. C# (게임, 웹)
6. TypeScript (웹 개발)
7. PHP (웹 서버)
8. Go (클라우드)
9. Rust (시스템)
10. Kotlin (안드로이드)

---

### Q2: 어떤 언어를 먼저 배워야 하나요?

**A**: 목적에 따라 다릅니다.

**입문용 추천**:
- **Python** (쉽고 다용도) ← 가장 추천
- **JavaScript** (즉시 웹에서 보기)
- **Scratch** (기초 개념, 어린이용)

**분야별 추천**:

| 분야 | 추천 언어 |
|------|----------|
| 웹 개발 | JavaScript, Python, PHP |
| 안드로이드 앱 | Kotlin, Java |
| iOS 앱 | Swift, Objective-C |
| 게임 | C#, C++, Rust |
| 머신러닝 | Python, R |
| 임베디드 (IoT) | C, C++, Rust |
| 데이터 분석 | Python, R, SQL |
| 클라우드 | Go, Python, Node.js |

---

### Q3: 컴파일 언어가 인터프리터 언어보다 항상 빠른가요?

**A**: 대부분 그렇지만, JIT 컴파일러를 쓰는 언어(Java, JavaScript V8)는 거의 비슷한 속도를 냅니다.

**최신 동향**:
- Java: JIT 컴파일로 C에 가까운 속도 달성
- JavaScript V8: 동적 최적화로 매우 빠름
- Python: PyPy 구현으로 속도 향상 (표준 CPython보다 5-10배 빠름)
- Rust: 메모리 안정성 + C 수준 속도

---

### Q4: 기계어를 직접 작성할 수 있나요?

**A**: 이론적으로는 가능하지만, 현실적으로는 너무 복잡합니다. 초기 컴퓨터 개발자들도 곧바로 어셈블리어로 넘어갔습니다.

**기계어 직접 작성이 어려운 이유**:
- 읽을 수 없음 (0과 1의 나열)
- 오류 찾기 불가능
- 수정 불가능
- 재사용 불가능

**대신 어셈블리어를 사용**:
- 사람이 읽을 수 있음
- 디버깅 가능
- 수정 가능

---

### Q5: AI가 프로그래머를 대체할까요?

**A**: AI는 도구일 뿐, 프로그래머의 역할은 더 중요해질 것입니다. AI에게 무엇을 만들지 지시하고, 결과를 검증하는 능력이 핵심이 됩니다.

**AI 시대의 프로그래머 역할**:

| 시대 | 프로그래머 역할 |
|------|-----------------|
| 과거 | 모든 코드를 직접 작성 |
| 현재 | 부분적으로 생성된 코드 검증 및 개선 |
| 미래 | 요구사항 정의 → AI 생성 → 검증 및 최적화 |

**필요한 기술**:
1. 프로그래밍 기초 이해 (이 가이드가 도움)
2. 문제 분석 및 설계 능력
3. AI 출력물 검증 능력
4. 새로운 기술 학습 능력

---

### Q6: 얼마나 공부하면 프로그래머가 될 수 있나요?

**A**: 개인차가 크지만, 대략적인 진행:

| 수준 | 시간 | 기간 |
|------|------|------|
| 기초 이해 (이 가이드) | 10-20시간 | 1-2주 |
| 한 가지 언어 기초 | 100시간 | 1-2개월 |
| 실무 초급 | 500시간 | 6-12개월 |
| 중급 | 2,000시간 | 2-3년 |
| 고급 | 5,000시간 | 5-10년 |

**팁**:
- 이론만 배우지 말고, 직접 만들기
- 자신의 프로젝트부터 시작
- 다른 개발자 코드 읽기
- 오픈소스 프로젝트 참여

---

### Q7: 프로그래밍 배울 때 가장 큰 실수는?

**A**: 다음 순서대로 하지 말 것:

1. ❌ 너무 어려운 언어부터 시작 (C++, Rust)
   - ✅ 쉬운 언어로 시작 (Python, Scratch)

2. ❌ 이론만 공부
   - ✅ 매일 직접 코드 작성

3. ❌ 완벽한 코드를 만들려고 시도
   - ✅ 작동하는 코드부터, 나중에 개선

4. ❌ 고립되어 혼자만 공부
   - ✅ 커뮤니티에 참여 (GitHub, Stack Overflow 등)

5. ❌ 너무 빨리 진행
   - ✅ 느리게, 탄탄하게 기초부터

---

### Q8: 프로그래밍 언어를 배우기 전에 수학을 배워야 하나요?

**A**: 높은 수준의 수학은 불필요합니다. 다만:

**필요한 수학**:
- 기본 사칙연산 ✅
- 변수 개념 ✅
- 간단한 논리 (if-else) ✅

**나중에 필요한 분야별 수학**:
- 게임 개발: 벡터, 삼각함수
- AI/머신러닝: 선형대수, 확률통계
- 과학 계산: 미적분, 방정식
- 암호화: 정수론, 이산수학

---

### Q9: 온라인 과정 vs 대학 학위, 어느 것이 낫나요?

**A**: 목표에 따라 다릅니다.

| 선택 | 장점 | 단점 |
|------|------|------|
| **온라인 과정** | 저렴, 빠름, 유연함 | 증명 없음, 자기관리 필요 |
| **대학 학위** | 증명 있음, 네트워크 | 비쌈, 시간 오래 걸림 |
| **부트캠프** | 집중적, 취업 지원 | 비쌈, 강도 높음 |

**추천**:
- 독학으로 시작 (이 가이드, CS50 등)
- 포트폴리오 만들기
- 필요시 온라인 인증서 추가
- 경력이 쌓이면 학위는 덜 중요

---

### Q10: 프로그래밍 공부하다 막혔을 때는?

**A**: 모든 프로그래머가 경험하는 일입니다. 당황하지 마세요!

**할 것들**:

1. **혼자 문제 해결 시도** (30분)
   - 코드 다시 읽기
   - 에러 메시지 검색
   - 디버깅 시도

2. **온라인 검색** (30분)
   - Stack Overflow 검색
   - 공식 문서 읽기
   - GitHub 유사 코드 찾기

3. **커뮤니티에 질문** (도움 받기)
   - Reddit r/learnprogramming
   - Stack Overflow (계정 필요)
   - 언어별 공식 커뮤니티

4. **다른 프로젝트로 이동**
   - 때로는 뇌를 쉴 필요
   - 다음 날 다시 시도하면 해결
