# 3D 소프트웨어 AI 통합 현황 (2025-2026)

노드 기반/코드 기반 3D 소프트웨어(Houdini, Blender)에 AI(LLM)를 연결하여 작업 효율을 높이는 현황 조사.

## 📖 핵심 아이디어

2025-2026년 현재, 3D DCC 도구와 AI의 통합은 **커뮤니티 주도**로 빠르게 진행 중이다. 공식 지원은 아직 제한적이지만, MCP 프로토콜과 LLM 코드 생성을 결합한 방식이 실용 단계에 진입했다. 핵심 흐름은 "AI가 기본 생성 → DCC 도구에서 정제"하는 하이브리드 워크플로우.

## 🔧 작동 방식 / 적용 방법

### Houdini MCP 동작 원리

```
Claude (MCP Client) ←→ MCP Server (Python) ←→ Houdini (hrpyc 소켓)
```

- Houdini 내부에 Python 소켓 서버 실행
- MCP 프로토콜로 Claude가 명령 전송
- 노드 생성, 파라미터 수정, 렌더링 트리거 등 가능
- 임의 코드 실행 리스크 주의

### Blender LLM 연동 방식

```
사용자 자연어 → LLM (코드 생성) → Blender Python API 실행
```

- Blender Python API가 완전 공개되어 LLM 코드 생성에 최적
- Function Call / MCP / 파인튜닝 등 다양한 접근

## 💡 실용적 평가 / 적용

### 트렌드 요약

| 방향 | 설명 | 성숙도 |
|------|------|--------|
| MCP 연동 | Claude/LLM이 직접 3D 소프트웨어 조작 | 초기 (커뮤니티) |
| 코드 생성 | 자연어→VEX/Python/노드그래프 | 실용 단계 |
| 메시 생성 AI | 텍스트/이미지→3D 모델 | 급성장 |
| 하이브리드 워크플로우 | AI 기본 생성→DCC 정제 | 2026 주류 |

### 후디니 vs 블렌더 비교

| | 후디니 | 블렌더 |
|--|--------|--------|
| AI 프로젝트 수 | 적음 (2~3개) | 많음 (10개+) |
| 공식 지원 | ML 추론 노드만 | 없음 |
| LLM 코드생성 난이도 | 어려움 (VEX+노드 복잡) | 쉬움 (Python API 깔끔) |
| 진입장벽 | 높음 (유료) | 낮음 (무료 오픈소스) |

### 한계

- 공식 지원은 양쪽 모두 없음 — 커뮤니티 주도
- Houdini MCP는 임의 코드 실행 보안 리스크
- LLM이 생성한 노드/코드의 품질은 아직 불안정
- 복잡한 프로시저럴 셋업은 LLM이 아직 어려워함
