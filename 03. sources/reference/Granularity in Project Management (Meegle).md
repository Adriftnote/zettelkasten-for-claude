# Granularity in Project Management (Meegle)

프로젝트 관리에서의 granularity 개념과 적정 수준 결정 방법을 정리합니다.

## 📖 핵심 정의

> "Granularity in project management refers to the level of detail and decomposition involved in breaking down project tasks and activities."

프로젝트를 더 작고 관리 가능한 구성요소로 **세분화하는 수준**

## 🛠️ 적정 수준 결정 방법

| 방법 | 설명 |
|------|------|
| **WBS** | Work Breakdown Structure - 계층적 분해 |
| **마일스톤** | 이해관계자에게 중요한 주요 지점 |
| **점진적 계획** | 초기 coarse → 진행하며 fine |
| **반복적 계획** | 스프린트마다 granularity 재평가 |

## 📊 Trade-off 분석

| 측면 | 높은 Granularity (Fine) | 낮은 Granularity (Coarse) |
|------|------------------------|--------------------------|
| 장점 | 정밀한 자원 배분, 위험 조기 감지 | 간단한 운영, 낮은 초기 비용 |
| 단점 | 관리 부담 증가, 분석 마비 위험 | 예측 정확도 낮음 |
| 유연성 | 변화 적응 어려움 | 변화에 유연함 |
| 팀 자율성 | 마이크로매니지먼트 위험 | 높은 자율성 |

## 💡 Agent Task 분해 시사점

| PM 개념 | Agent 적용 |
|--------|-----------|
| WBS | 계층적 Task 분해 (Level 0 → 1 → 2) |
| 마일스톤 | 중간 검증점, Attention Anchor |
| 분석 마비 | 너무 많은 subtask → 조정 오버헤드 > 실행 가치 |
