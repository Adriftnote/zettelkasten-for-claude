# TikTok 저장수(Saves)와 알고리즘 장수 컨텐츠 관계

TikTok 알고리즘에서 저장(Saves/Bookmark)이 컨텐츠 장수에 미치는 영향을 분석한 조사 결과와 실증 데이터.

## 📖 핵심 아이디어

2025년 TikTok 알고리즘 업데이트에서 저장(Saves)과 공유(Shares)에 추가 가중치가 부여되었다. 저장은 "다시 볼 의향"의 신호이며, 재방문 시 완주율(1위 시그널)에도 기여하여 알고리즘 노출이 장기간 유지되는 선순환 구조를 만든다. 인스타그램과 동일한 방향의 변화.

## 🛠️ 알고리즘 가중치 순위 (2025-2026)

| 순위 | 지표 | 가중치 | 설명 |
|------|------|--------|------|
| 1 | 완주율/시청시간 | 최고 | 영상을 끝까지 보는 비율 |
| 2 | 저장(Saves) + 공유(Shares) | 높음 | "meaningful engagement"로 분류 |
| 3 | 댓글 | 중간 | 적극적 참여 |
| 4 | 좋아요 | 낮음 | "shallow engagement" |

## 🔧 저장 → 장수 선순환 가설

```
저장(Bookmark) → 재방문 의도 신호
    → 알고리즘이 신규 유저에게 추가 노출
    → 신규 유저 완주 → 완주율 유지
    → 추가 노출 지속 (선순환)
```

저장이 공유보다 더 강한 영향을 미치는 이유:
- 공유: 타인에게 전달하는 행위 (1회성)
- 저장: 본인이 재시청하겠다는 의도 → 완주율 재기여 가능성 높음

## 💡 실증 데이터 (쎈정보 TikTok 계정)

2월 정합성 검증에서 초기 25개 영상 중 수집이 끊긴 후에도 조회수가 계속 증가한 영상 2개 발견:

| video_id | 제목 | 업로드일 | 마지막 수집 시 조회수 | 2일 증가 | 특징 |
|----------|------|---------|---------------------|---------|------|
| 7575454311535152402 | 아이폰 K-패스 사용법 | 2025-11-22 | 331 | +10 | 실용 정보 |
| 7566829836010835207 | 데이터 아끼는 설정 | 2025-10-30 | 139 | +139 (데이터 이상) | 실용 정보 |

나머지 23개 영상은 수집 중단 시점에 조회수 증가 0 (자연사). 두 영상 모두 실용 정보 컨텐츠로 저장(북마크) 발생 유형.

## 🔧 TikTok API 저장수 수집 방법

TikTok 공식 API(v2 video/query)에는 저장수 필드가 없음:

| API/방법 | 필드명 | 접근성 |
|----------|--------|--------|
| v2 video/query (공식) | 없음 | 우리가 사용 중 |
| Research API (공식) | `favorites_count` | 학술/비영리만 허용 |
| 웹 스크래핑 | `collectCount` | `__UNIVERSAL_DATA_FOR_REHYDRATION__` JSON에서 추출 |
| yt-dlp (리버스엔지니어링) | `collect_count` | 웹 스크래핑과 동일 원리 |
