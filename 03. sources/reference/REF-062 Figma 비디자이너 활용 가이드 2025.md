# Figma 비디자이너 활용 가이드 2025

Figma는 디자인 도구이지만, 2025년 기준 비디자이너의 56%가 디자인 관련 작업을 직접 처리하는 추세. PM의 70%가 와이어프레임/목업을 직접 제작 중.

## 📖 핵심 아이디어

Figma는 "디자이너의 VS Code" — 브라우저 기반 클라우드 디자인 도구로, 실시간 협업이 내장. 비디자이너에게는 **시각적 커뮤니케이션 도구**로 확장되고 있으며, 특히 FigJam(화이트보드)과 Figma Slides(발표자료)가 진입 장벽을 낮춤.

## 🛠️ 직군별 활용 사례

| 직군 | 활용 방식 | 구체적 예시 |
|------|----------|------------|
| PM/기획자 | 와이어프레임, 스펙 문서 통합 | 기획서+디자인을 한 파일에 (카카오워크팀 사례) |
| 개발자 | Dev Mode, 아키텍처 다이어그램 | CSS/토큰 자동 추출, 시스템 구조도 |
| 마케터 | 배너/SNS 이미지 제작 | Figma Buzz로 템플릿 기반 일괄 생성 |
| 데이터/자동화 | 파이프라인 문서화, API 시각화 | FigJam으로 워크플로우 전체 그림 |
| 누구든 | 브레인스토밍, 회고 | FigJam 포스트잇 + AI 요약 |

## 🔧 주요 기능별 용도

### Figma Design (본 에디터)
- UI 시안 제작 (디자이너 메인 도구)
- Dev Mode: 개발자가 간격, 색상, 폰트 수치 직접 확인
- 컴포넌트 라이브러리: 재사용 가능한 버튼, 카드 등 관리
- 프로토타이핑: 클릭 인터랙션 시뮬레이션

### FigJam (화이트보드) — 비디자이너 진입점
- 스프린트 회고: 팀원 각자 포스트잇 → 투표 → 우선순위
- 시스템 설계: 화살표로 데이터 흐름 도식화
- 브레인스토밍: AI 자동 그룹화/요약 (2025~)
- 의사결정 트리, 고객 여정 지도

### Figma Slides (발표자료)
- PPT 대용, 디자인 컴포넌트 그대로 삽입 가능
- 3,400개+ 무료 템플릿
- 단점: 오프라인 불안정, 저장 오류 보고 있음

### Figma MCP (AI 코딩 연동)
- Figma → Code: `get_design_context`로 디자인 시안을 코드로 변환
- Code → Figma: `generate_figma_design`으로 웹 페이지를 Figma 레이어로 캡처
- VS Code, Cursor, Claude 등 AI 코딩 도구와 연동

### Figma Buzz (2025 신규)
- 디자이너가 템플릿 잠금 설정 → 비디자이너는 텍스트/이미지만 교체
- 스프레드시트 데이터 연동으로 다국어 배너 일괄 생성

## 💡 실용적 평가 / 적용

### 자동화/백엔드 엔지니어에게 추천하는 활용

**즉시 가능 (학습 1~2시간):**
1. FigJam으로 n8n 워크플로우 전체 흐름도 문서화
2. Chrome 확장프로그램 UI를 코드 전에 먼저 스케치
3. 대시보드 레이아웃 와이어프레임으로 사전 확인받기

**학습 후:**
4. Figma MCP + AI로 디자인 → 코드 자동 변환
5. Figma Slides로 데이터 분석 보고서

### 대안 도구와 비교
- 다이어그램: Mermaid, draw.io, LikeC4로 대체 가능
- 화이트보드: Miro도 유사 기능
- 차별점: Figma는 디자인 + 협업 + 프로토타입이 한 도구에 통합

### 현실적 판단
백엔드/자동화 위주라면 "없으면 안 되는 도구"는 아님. UI 포함 작업(확장프로그램, 대시보드)에서 협업 시 실용적. FigJam은 진입장벽 최저.
