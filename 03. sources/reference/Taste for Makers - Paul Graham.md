# Taste for Makers - Paul Graham

미감(taste)은 주관적 취향이 아니라, 무엇이 더 나은지 구분하고 만들어낼 수 있는 실질적 능력이다.

## 📖 핵심 아이디어

Paul Graham은 "좋은 디자인"이 분야를 막론하고 공통 원칙을 가진다고 주장한다. 수학의 증명, 과학 이론, 그림, 건축, 소프트웨어 모두 같은 아름다움의 기준을 공유하며, 이 미감은 **배울 수 있는 능력**이다. 위대한 작업의 비결은 "극도로 까다로운 미감 + 그것을 만족시킬 능력"의 결합.

## 🛠️ Good Design의 14가지 원칙

| # | 원칙 | 핵심 |
|---|------|------|
| 1 | **단순하다** (Simple) | 핵심에 집중, 불필요한 장식 제거 |
| 2 | **시대를 초월한다** (Timeless) | 유행이 아닌 영속적 가치 추구. "과거에 어필하면 미래에도 어필" |
| 3 | **올바른 문제를 푼다** (Right Problem) | 기술적 가능성이 아닌 사용자 필요 우선 |
| 4 | **암시적이다** (Suggestive) | 완전한 설명보다 상상의 여지를 남김 |
| 5 | **약간 유머가 있다** (Slightly Funny) | 자신을 너무 심각하게 여기지 않는 자신감 |
| 6 | **어렵다** (Hard) | 난제와의 투쟁이 우아한 해법을 낳음. 제약이 불필요한 요소를 제거 |
| 7 | **쉬워 보인다** (Looks Easy) | 복잡한 노력을 감추고 자연스럽게. 8번 수정 후의 대화체 |
| 8 | **대칭적이다** (Symmetry) | 반복과 재귀를 통한 단순화. 에펠탑 = "탑 위의 탑" |
| 9 | **자연을 닮는다** (Resembles Nature) | 자연은 오랜 시간의 문제 해결 결과 |
| 10 | **재설계이다** (Redesign) | 완벽한 첫 시도는 없음. 반복 개선 |
| 11 | **모방할 수 있다** (Can Copy) | 무의식적 모방 → 의도적 독창성 → "정확성 > 독창성" 깨달음 |
| 12 | **낯설다** (Often Strange) | 기묘한 아름다움은 의도가 아닌 올바름 추구의 부산물 |
| 13 | **무리 지어 일어난다** (Happens in Chunks) | DNA보다 환경·커뮤니티가 중요 (피렌체 vs 밀라노) |
| 14 | **대담하다** (Often Daring) | 관습에 어긋나는 시도가 혁신. 코페르니쿠스는 못생긴 이론이 불편해서 새 이론 추구 |

## 🔧 실천적 함의

```
좋은 만듦의 공식:

   못생김에 대한 불관용 (Intolerance for Ugliness)
 + 분야에 대한 깊은 이해 (Deep Domain Knowledge)
 + 반복적 개선 (Iterative Redesign)
 = 위대한 작업
```

- **단순함을 추구하되 단순주의에 빠지지 말 것** — 핵심을 드러내는 단순함이지, 내용을 버리는 단순함이 아님
- **시대초월 테스트** — "30년 전에도 좋아 보였을까?" 그렇다면 30년 후에도 좋을 가능성 높음
- **쉬워 보이게 만드는 건 어려운 일** — 숙련의 증거는 노력이 안 보이는 것
- **낯설음을 의도하지 말 것** — 올바른 답을 추구하면 자연스럽게 따라옴

## 💡 실용적 평가 / 적용

**소프트웨어/코드에 적용:**
- 단순하다 → 불필요한 추상화 제거, 최소 코드
- 올바른 문제를 푼다 → 기능 과잉(over-engineering) 경계
- 재설계이다 → 첫 구현에 집착하지 말고 리팩터링
- 쉬워 보인다 → 잘 설계된 API는 사용자가 복잡성을 모름

**글의 한계:**
- 2002년 에세이로, 구체적 사례가 다소 오래됨
- "좋은 디자인"의 보편성 주장은 문화적 맥락을 단순화하는 면이 있음
