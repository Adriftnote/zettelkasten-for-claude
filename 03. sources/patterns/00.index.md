# 디자인 패턴 & 최적화 전략 인덱스

**루만 제텔카스텐 원칙**으로 정리된 설계 패턴과 최적화 전략 모음입니다. 각 패턴이 독립적인 노트로 분리되어, 상황에 맞게 참고할 수 있습니다.

이 폴더의 패턴들은 실제 개발/운영 경험에서 도출된 **검증된 해법들**입니다.

---

## 📚 패턴 목록

### 1️⃣ Context Engineering 패턴 (7개)

LLM의 컨텍스트 최적화와 관련된 패턴들입니다.

- [[progressive-disclosure|Progressive Disclosure]] - 필요할 때만 정보 로드
- [[lost-in-middle|Lost-in-Middle]] - 컨텍스트 중간 정보의 저하 현상
- [[context-poisoning|Context Poisoning]] - 오류 누적 현상
- [[context-distraction|Context Distraction]] - 관련 없는 정보의 방해
- [[context-confusion|Context Confusion]] - 작업 간 문맥 혼동
- [[context-clash|Context Clash]] - 상충하는 정보의 공존
- [[four-bucket-optimization|Four-Bucket Optimization]] - 4가지 최적화 전략
- [[observation-masking|Observation Masking]] - 도구 출력 압축
- [[anchored-iterative-summarization|Anchored Iterative Summarization]] - 반복적 요약 압축

### 2️⃣ MCP/Tool 패턴 (5개)

MCP (Model Context Protocol) 및 도구 설계 패턴들입니다.

- [[lazy-tool-loader|LazyToolLoader Pattern]] - 캐시 기반 지연 로딩
- [[dynamic-tool-fetching|Dynamic Tool Fetching]] - 런타임 도구 검색
- [[hook-based-mcp-auto-activation|Hook-based MCP Auto-Activation]] - 키워드 기반 자동 활성화
- [[tool-discovery-pattern|Tool Discovery Pattern]] - 도구 검색 및 등록
- [[consolidation-principle|Consolidation Principle]] - 도구 통합 원칙

### 3️⃣ 최적화 전략 (3개)

성능과 안정성 관련 최적화 패턴들입니다.

- [[token-optimization-strategy|Token Optimization Strategy]] - 토큰 사용 최적화
- [[kv-cache-optimization|KV-Cache Optimization]] - 키-값 캐시 최적화
- [[graceful-degradation|Graceful Degradation]] - 우아한 성능 저하

### 4️⃣ Metabase 패턴 (5개)

Metabase BI 도구 관련 패턴들입니다.

- [[dropdown-filter-pattern|Dropdown Filter Pattern]] - 드롭다운 필터 구현
- [[model-configuration-pattern|Model Configuration Pattern]] - 모델 메타데이터 설정
- [[server-selection-pattern|Server Selection Pattern]] - Metabase 서버 선택
- [[text-card-addition-pattern|Text Card Addition Pattern]] - 텍스트 카드 추가
- [[using-variables-in-card-titles|Using Variables in Card Titles]] - 카드 제목의 변수 활용

1. **상황 기반 선택**: 위 "패턴 선택 가이드"에서 상황에 맞는 패턴 찾기
