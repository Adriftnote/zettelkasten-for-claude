# 초기 노트의 범위 과잉이 하류 노트로 전파된다

## 문제 상황

04. notes 전수 검토 중 탈락 노트 2건과, 해당 노트를 참조하는 하류 문서를 발견했다.

탈락 노트:
- **Fast-Slow 프랙탈 - 도메인을 관통하는 구조** (2026-01-20) — 카너먼 시스템1/2, 캐시/메모리, Engram, KGGen을 "Fast-Slow"로 묶었으나 각각 원인이 다름. "빠른 것과 느린 것이 있다"는 거의 모든 시스템에 적용 가능한 너무 넓은 패턴.
- **Engram과 지식 구조 - KGGen 비교** (2026-01-20) — "정적-동적 분리"로 3개 시스템을 묶었으나 동일한 문제. Fast-Slow 프랙탈에서 derived_from하여 같은 패턴을 계승.

이 탈락 노트를 참조하는 하류 문서:

```
Fast-Slow 프랙탈 (탈락)
  ├→ hub: Fast-Slow 인지 패턴 — organizes 관계
  ├→ hub: 컨텍스트-메모리 통합 — references + 섹션 전체
  └→ note: Three-Layer Memory Architecture — "이론적 기반"으로 6회 참조

Engram과 지식 구조 (탈락, 프랙탈에서 derived_from)
  ├→ note: 심볼릭 AI vs 커넥셔니스트 AI 역사 — derived_from
  ├→ note: Fast-Slow 프랙탈 — derived_from (상호 참조)
  └→ note: Three-Layer Memory Architecture — 비교 분석 참조
```

## 근본 원인

초기 노트(Fast-Slow 프랙탈)가 범위 과잉(4개 도메인 강제 통합)으로 작성되었고, 이후 노트들이 이를 derived_from으로 참조하면서 잘못된 전제가 전파되었다. 다중 홉 추론에서 첫 홉이 틀리면 이후 홉도 틀리는 구조와 동일하다.

프랙탈 노트는 vault 초기에 작성되어 3원칙이 아직 수립되기 전이었다.

## 해결책

1. 탈락 노트 2건 삭제 (04. notes에서 제거)
2. 하류 문서는 개별 검토 필요 — 탈락 노트 참조를 제거하거나 독립 근거로 교체
   - Three-Layer Memory Architecture: 프랙탈 참조 6건 제거 필요
   - 심볼릭 AI vs 커넥셔니스트 AI 역사: Engram derived_from 제거, 독립 근거로 성립 가능한지 검토
   - hub 2건: 관련 섹션 정리 필요

## 적용

- Fast-Slow 프랙탈, Engram 비교 노트 삭제 완료
- 하류 문서 정리는 별도 작업으로 진행 예정

## Observations

- [fact] derived_from 관계가 잘못된 전제를 하류 노트로 전파한다 — 다중 홉에서 첫 홉 오류와 동일한 구조 #전파
- [fact] 초기 노트(3원칙 수립 전)가 가장 위험하다 — 기준 없이 작성되었으면서 이후 노트의 근거로 쓰이기 때문 #초기노트
- [warning] 범위 과잉 노트가 hub나 아키텍처 문서에 "이론적 기반"으로 참조되면 영향 범위가 급격히 넓어진다 #확산위험
