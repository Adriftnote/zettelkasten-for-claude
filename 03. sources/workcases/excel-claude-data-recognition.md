# Excel Claude 데이터 인식 방식

## 개요

Excel Claude가 스프레드시트 데이터를 어떻게 인식하는지 정리.

---

## 인식 방법 2가지

| 방법 | 용도 | 특징 |
|------|------|------|
| **CSV 형식** (get_range_as_csv) | 데이터 분석, 대량 처리 | 값만, 서식/수식 미포함 |
| **셀 범위 상세** (get_cell_ranges) | 수식/서식 확인 | JSON, 메모까지 포함 |

---

## 일반 Claude vs Excel 통합 Claude

| 구분 | 일반 Claude | Excel 통합 Claude |
|------|------------|------------------|
| 인식 방식 | 텍스트/CSV로 복사-붙여넣기 | API를 통해 직접 엑셀 파일 접근 |
| 수식 확인 | X (결과값만 보임) | O (수식 원본 확인 가능) |
| 서식 확인 | X | O (색상, 폰트 등 확인 가능) |
| 데이터 수정 | X | O (직접 셀에 쓰기 가능) |

---

## 인식 가능 여부

| 항목 | 읽기 | 쓰기 |
|------|:----:|:----:|
| 셀 데이터 (텍스트, 숫자) | O | O |
| 수식 및 계산 결과 | O | O |
| 셀 서식 (색상, 폰트, 테두리) | O | O |
| 병합된 셀 | **X** | **X** |
| 메모/노트 | O | O |
| 피벗 테이블 | O | O |
| 차트 | O | O |
| **이미지/도형** | **X** | **X** |

---

## 핵심 한계

> **셀 단위로 데이터를 읽기 때문에, 전체 레이아웃을 "한눈에" 보는 건 어려움**

| 할 수 있는 것 | 어려운 것 |
|--------------|----------|
| 각 셀의 값, 수식, 서식 읽기 | 표의 시각적 전체 모습 파악 |
| 데이터 패턴 분석 | "이 표가 예쁘게 정렬됐는지" 판단 |
| 행/열 숨기기/보이기 | **병합 셀 읽기/작업** |

---

## 병합 셀 인식 한계

> [!warning] 병합 정보는 읽을 수 없음
> `get_cell_ranges` API는 값, 수식, 서식, 메모를 반환하지만 **병합 셀 정보는 포함하지 않음**

### 실제 사례

| 항목 | Claude가 인식한 것 | 실제 (이미지 기준) |
|------|-------------------|-------------------|
| C8 값 | ✅ 있음 | 있음 |
| D8, E8, F8 값 | 비어있음 | **C8과 병합됨** |
| 병합 여부 | ❌ 모름 | 병합되어 있음 |

### 결론

- **병합 읽기**: ❌ 불가능 (어떤 셀이 병합되어 있는지 확인 불가)
- **병합 작업**: ❌ 지원 안 됨
- **대안**: 스크린샷/이미지로 시각적 확인 필요

## 전체 한계점 정리

### 시각적 요소

| 항목 | 상태 |
|------|:----:|
| 이미지/그림 | ❌ |
| 도형 (Shape) | ❌ |
| 아이콘 | ❌ |
| 스파크라인 | ❌ |
| 조건부 서식 규칙 | ❌ (결과 색상만 보임, 규칙 자체는 못 읽음) |

### 고급 기능

| 항목 | 상태 |
|------|:----:|
| 데이터 유효성 검사 (드롭다운 등) | ❌ |
| 하이퍼링크 | ❌ |
| 필터/정렬 상태 | ❌ |
| 그룹화 (행/열 접기) | ❌ |
| 슬라이서 | ❌ |
| 매크로/VBA | ❌ |

### 레이아웃 관련

| 항목 | 상태 |
|------|:----:|
| 인쇄 영역 설정 | ❌ |
| 페이지 나누기 | ❌ |
| 머리글/바닥글 | ❌ |

### 기타

| 항목 | 상태 |
|------|:----:|
| 숨겨진 행/열 읽기 | ❌ (숨기기/보이기 작업은 가능) |
| 시트 보호 상태 | ❌ |
| 이름 정의 (Named Ranges) 목록 | ❌ |

## 잘 할 수 있는 것

> [!tip] Excel Claude 강점
> - 셀 값, 수식, 수식 결과
> - 서식 (폰트, 색상, 테두리, 정렬)
> - 메모/노트
> - 피벗 테이블, 차트 생성/수정
> - 행/열 추가, 삭제, 숨기기
> - 시트 생성/삭제/이름변경

---

## 시사점

1. **프롬프트에 구조 설명 필요**: 전체 레이아웃을 못 보니까 "B열은 담당자, G열은 내용" 같이 명시
2. **양식 이미지 첨부 권장**: 시각적 맥락 제공 (특히 병합 셀 확인용)
3. **병합 정보 명시 필요**: 병합 상태를 읽을 수 없으므로 "C8:F8은 병합됨" 같이 알려줘야 함
4. **이미지/도형 작업 불가**: 별도 처리 필요
