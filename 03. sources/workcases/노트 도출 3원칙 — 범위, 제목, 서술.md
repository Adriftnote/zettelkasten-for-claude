# 노트 도출 3원칙 — 범위, 제목, 서술

> 도출 노트 3회 작성(실패 2 + 성공 1)에서 품질을 결정하는 3가지 원칙을 추출

## 1. 전체 흐름

```
노트 3회 작성 → 실패 2건 분석 → 성공 1건 비교 → 공통 패턴 추출 → 3원칙 도출
```

```
[실패1: 캐시는 어디에나 있다] ──→ 표면적 비유로 4개 도메인 강제 통합
[실패2: Big O는 메모리를 모른다] ──→ 제목과 내용 불일치 + 거짓 서술
[성공: 연결리스트를 쓰지 말아야 하는 이유] ──→ 좁은 범위, 사실 기반, ★★★★★
                    ↓
              3원칙: 범위 · 제목 · 서술
```

## 2. 문제 상황

도출 노트 작성 시 품질이 일관되지 않았다. 제텔카스텐 Notes 갭 분석 이후 notes 도출을 시도했으나, 3개 중 2개가 실패했다.

- **실패 빈도**: 3회 중 2회 (67%)
- **실패 유형**: 범위 초과, 제목-내용 괴리, 사실 아닌 서술
- **영향**: 노트 신뢰성 저하, 작성 시간 낭비

## 3. 실패/성공 분석

### 3-1. 실패: "캐시는 어디에나 있다"

- CPU 캐시, KV 캐시, Progressive Disclosure, 시스템1을 "캐시"로 묶음
- 공유하는 건 "가까이 둔다"는 **표면적 비유**뿐, 메커니즘은 전부 달랐다
- "캐시의 보편성은 캐시 실패의 보편성을 의미한다" — 의미 없는 문장
- **근본 원인**: 범위를 비유 수준에서 잡음. 메커니즘 공유 여부를 확인하지 않음

### 3-2. 실패: "Big O는 메모리를 모른다"

- 내용은 연결리스트가 배열에 지는 이유인데, 제목은 추상적 주장
- "교과서에서 이렇게 가르친다"는 서술 — 본인이 그렇게 배운 적 없음
- **근본 원인 1**: 제목이 내용을 반영하지 않음 (멋있어 보이는 제목 선호)
- **근본 원인 2**: 사실이 아닌 것을 그럴싸하게 포장

### 3-3. 성공: "연결리스트를 쓰지 말아야 하는 이유"

- 배열, 연결리스트, 캐시 라인이 **하나의 물리적 원인**(캐시 라인 64B 프리페치)으로 연결
- 제목이 곧 내용
- 벤치마크 20배 + Rust 공식 비권장이라는 **사실만** 기술
- 평가: ★★★★★

## 4. 도출된 원칙

### 원칙 1: 범위 — 메커니즘이 같은 것끼리만

- 연결하려는 개념들이 **하나의 원인**을 공유해야 한다
- "이 둘이 같은 이유"를 한 문장으로 못 하면 억지 연결
- 표면적 비유 공유 ≠ 메커니즘 공유

### 원칙 2: 제목 — 읽으면 내용이 보여야 함

- 제목만 보고 "무슨 얘기인지" 바로 알 수 있어야 한다
- 추상적이거나 멋있어 보이는 제목은 내용과 어긋난다

### 원칙 3: 서술 — 사실만, 포장 없이

- 본인이 모르거나 경험하지 않은 것을 아는 척 쓰지 않는다
- 그럴싸하게 들리지만 의미 없는 문장은 쓰지 않는다
- 사실 + 원인, 끝

## 5. 적용 방법

노트 작성 전/중/후 3단계 자문:

1. **작성 전** (범위): "이 개념들이 같은 이유를 한 문장으로 말할 수 있는가?"
2. **제목 작성 후** (제목): "이 제목만 보고 내용을 알 수 있는가?"
3. **서술 완료 후** (서술): "이게 사실인가, 포장인가?"

## Observations

- [pattern] 도출 노트 실패의 67%는 범위 초과에서 시작된다 — 비유가 아닌 메커니즘 공유 여부가 핵심 #범위검증
- [pattern] 멋있어 보이는 제목일수록 내용과 괴리될 확률이 높다 #제목검증
- [pattern] 있어 보이는 문장은 의미가 없을 확률이 높다 — 그럴싸함은 사실이 아님의 신호 #서술검증
- [fact] 4개 도메인 강제 통합보다 2-3개 concept의 좁은 연결이 더 나은 노트를 만든다 #실패사례
- [method] 노트 작성 전 "이 개념들의 공통 메커니즘을 한 문장으로 말할 수 있는가?" 자문한다 #품질체크리스트
- [method] 제목을 쓴 뒤 "이 제목만 보고 내용을 알 수 있는가?" 확인한다 #품질체크리스트
- [method] 서술을 쓴 뒤 "이게 사실인가, 포장인가?" 구분한다 #품질체크리스트
