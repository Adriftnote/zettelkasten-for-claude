# EMPO² 논문에서 소버린 AI까지 — AI 안전성 관점 지식 도출 과정

> REF-074 EMPO² 논문 리뷰에서 출발하여, AI 안전성 위험 분석 → 소버린 AI 필요성까지 지식을 도출한 과정 기록. 노트 도출 3원칙이 가설적 아이디어를 걸러낸 성공 사례 포함.

## 1. 전체 흐름

```
EMPO² 논문 리뷰 (REF-074)
    │
    ├─ 기술 이해: 자가 생성 메모리 → 파라미터 증류
    │
    ├─ 위험 발견: 이 메커니즘이 왜곡을 영구화할 수 있다
    │       │
    │       └─ 기존 concepts 조합:
    │           Context Poisoning (자기강화 루프)
    │           + 표면적 정렬 (감사 불가)
    │           = note 도출 ✅
    │
    ├─ 확장 1: 폐쇄형 AI 사용자 입장 → 편향 인식 필요성
    │
    ├─ 확장 2: 국가 단위 → 소버린 AI 필요성
    │       └─ concept 생성 ✅
    │
    └─ 확장 3: 월드 모델 + 국가별 증류 아이디어
            └─ 3원칙 검증 → 서술 원칙 미통과 → 보류 ✅
```

## 2. 생성된 산출물

### 2-1. note: 자가 생성 메모리의 파라미터 내재화는 왜곡을 영구화한다

- **도출 근거**: EMPO² 증류 메커니즘 + Context Poisoning + 표면적 정렬
- **핵심 결론**: 왜곡된 팁이 파라미터에 증류되면 (1) 세션 초기화로도 복구 불가 (2) 외부 감지 불가 (3) 데이터 감사 불가
- **3원칙 통과**: 범위(공통 메커니즘: 자기강화 루프의 비가역적 내재화) ✅, 제목 ✅, 서술(논문 사실 기반) ✅

### 2-2. concept: 소버린 AI (Sovereign AI)

- **정의**: 국가가 자체 인프라·데이터·인력·네트워크로 AI를 독자 구축·통제하는 역량
- **구성**: NVIDIA 정의 4요소 (물리 인프라, 데이터 인프라, 인력, 비즈니스 네트워크)
- **안전성 연결**: 폐쇄형 AI 의존 시 표면적 정렬 문제가 국가 단위로 확장됨

### 2-3. 보류: 월드 모델 + 국가별 증류 아이디어

- **아이디어**: UN 같은 국제 월드 모델을 공유 기반으로, 각국이 EMPO² 증류로 소버린 AI 구축
- **3원칙 검증 결과**:
  - 범위 ⚠️ — 기술 메커니즘과 정책 적용의 추상화 레벨이 다름
  - 제목 ✅ — 명확
  - 서술 ❌ — 핵심 결론이 미검증 가설 ("~일 수 있다")
- **판정**: note 생성 보류, 근거 축적 후 재검토

## 3. 학습 포인트

### 논문 → 위험 분석의 전환

EMPO²를 단순히 "성능 좋은 RL 기법"으로 읽지 않고, "이 메커니즘이 악용되면?"이라는 질문을 던진 것이 위험 분석 note 도출의 출발점이었다. 기술 논문 리뷰 시 "반대 방향"을 항상 고려하는 것이 유효한 패턴.

### 3원칙의 실전 작동

"월드 모델 + 국가별 증류"는 대화에서 자연스럽고 설득력 있게 느껴졌지만, 3원칙 서술 검증에서 "아직 사실이 아닌 가설"이라는 점이 드러났다. 매력적인 아이디어일수록 3원칙 검증이 중요하다.

### 도출 과정의 자연스러운 확장

```
기술 이해 → 위험 발견 → 개인 수준 시사점 → 국가 수준 시사점 → 해결 아이디어
```

각 단계가 자연스럽게 이어졌고, 각 단계에서 적절한 노트 유형(note, concept)을 선택한 것이 볼트 구조에 맞았다.

## Observations

- [pattern] 기술 논문에서 "이 메커니즘이 악용되면?"을 질문하면 안전성 note가 도출된다 #논문리뷰패턴
- [pattern] 대화에서 자연스럽게 확장된 아이디어일수록 3원칙 검증이 필수적이다 #3원칙
- [fact] 3원칙 서술 원칙이 가설적 아이디어(월드 모델+국가별 증류)를 걸러냈다 #품질관리
- [method] 노트 유형 선택 기준: 사실 기반 도출 → note, 개념 정의 → concept, 미검증 가설 → 보류 #노트유형판단
- [fact] EMPO²의 핵심 위험은 Context Poisoning의 파라미터 레벨 확장으로 프레이밍할 수 있다 #AI안전성
