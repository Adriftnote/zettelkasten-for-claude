# Excel Claude 프롬프트 vs 작업명세서 효율 비교

## 배경

주간보고서 작성을 Excel Claude에 위임할 때, 두 가지 방식 중 어떤 것이 더 효율적인지 테스트.

- **프롬프트 방식**: 자연어로 작업 지시 + 양식 이미지 첨부
- **작업명세서 방식**: CSV/표 형태로 셀 주소와 내용 명시

---

## 테스트 환경

### 입력 데이터
- Week 05 (1/19-23) 완료 작업 11건
- 금주 계획 3건
- 담당자: 유정우

### 생성 파일

| 파일 | 형식 | 크기 |
|------|------|------|
| `week05-prompt.md` | 프롬프트 | 2,778 bytes |
| `week05-spec.csv` | 작업명세서 | 3,363 bytes |

## 프롬프트 형식 예시

```
주간보고서 양식에 넣어줘.
- B열: 담당자
- C열: 업무명
- G열: 업무진행내용 (작은따옴표 붙여서 입력)
- 화살표는 -> 로
- 지난주 업무: Row 8부터
- 금주 계획: Row 17부터

[이번 주 완료 업무]

1. [자동화] Flow Task Creator 사이드패널 전환
   - 팝업 UI -> 사이드 패널 전환으로 UX 개선
   - Task 생성 탭 API 개선, 그룹 선택 기능 추가
...
```

## 작업명세서 형식 예시

```csv
작업 ID,작업 유형,대상 시트,범위,내용,비고,상태
1,데이터 입력,주간보고,B8,유정우,담당자,대기
2,데이터 입력,주간보고,C8,[자동화] Flow Task Creator 사이드패널 전환,업무명,대기
3,데이터 입력,주간보고,G8,'팝업 UI -> 사이드 패널 전환...,진행내용,대기
...
```

업무 1건당 3행 필요 (담당자/업무명/내용 분리)

## 토큰 효율 비교

| 항목 | 프롬프트 | 명세서 |
|------|----------|--------|
| **파일 크기** | 2,778 bytes | 3,363 bytes |
| **업무당 크기** | ~253 bytes | ~374 bytes |
| **효율** | **21% 더 효율적** | 기준 |

### 명세서가 큰 이유
- 매 행마다 반복되는 메타데이터 (작업 ID, 작업 유형, 대상 시트, 상태)
- 담당자가 매 행마다 반복
- 셀 주소(B8, C8, G8) 하드코딩

### 결과물 품질 비교

| 항목 | 프롬프트 + 이미지 | 명세서 (CSV) |
|------|------------------|--------------|
| **업무 수** | 11건 (전체) | 9건 (2건 누락) |
| **금주 계획** | 3건 작성됨 | 빈칸 |
| **레이아웃** | 깔끔하게 정렬 | 텍스트 넘침, 잘림 |
| **줄바꿈** | 자동 적용 | 안됨 |
| **셀 높이** | 내용에 맞게 조정 | 고정 (내용 잘림) |
| **행 추가** | 부족하면 자동 추가 | 불가 |

## 핵심 발견

### 프롬프트 방식이 우월한 이유

Excel Claude가 **이미지를 볼 수 있기 때문**:

1. **표 구조 인식**: "B열이 담당자, G열이 내용이구나" 시각적으로 이해
2. **자동 서식 조정**: 셀 너비, 높이, 줄바꿈 알아서 적용
3. **유연한 대응**: 행 부족하면 자동 추가
4. **양식 변경 대응**: 표가 바뀌어도 알아서 적응

### 명세서 방식의 한계

1. **하드코딩 의존**: `B8`, `G8` 같은 셀 주소에 의존
2. **표 구조 무지**: 병합 셀, 레이아웃 파악 불가
3. **서식 별도 필요**: 열 너비, 줄바꿈 등 추가 명령 필요
4. **양식 변경 취약**: 주소 전부 수정해야 함

## 결론

| 관점 | 추천 형식 |
|------|----------|
| **토큰 효율** | 프롬프트 (21% 절약) |
| **결과 품질** | 프롬프트 (누락 없음, 서식 깔끔) |
| **작업 턴 수** | 프롬프트 (2턴 완료) |
| **유연성** | 프롬프트 (자동 행 추가, 서식 조정) |
| **유지보수** | 프롬프트 (양식 변경에 강함) |

**최종 결론**: Excel Claude처럼 **이미지 볼 수 있는 환경**에서는 **프롬프트 + 양식 이미지** 방식이 압도적으로 효율적.

명세서 방식은 API 자동화나 이미지 못 보는 환경에서만 의미 있음.
