# note 3원칙 실패 사례 — 비유 공유를 메커니즘 공유로 착각

## 문제 상황

Search-R1++ 논문 대화에서 "정보 수집은 빠르게, 의미 도출에서만 깊게"라는 노트를 /note 스킬로 생성했다. 다음 두 개념을 연결하려 했다:

- Search-R1++의 Fast Thinking > Slow Thinking (AI 검색 에이전트 실험 결과)
- 제텔카스텐 워크플로우: source→concept은 빠르게, concept→note는 깊게

## 근본 원인

**원칙 1 (범위) 위반**: 두 개념이 공유하는 건 "빠르게/깊게"라는 비유뿐이었다.

- Search-R1++에서 Fast가 이긴 이유: `<think>` 태그가 보상 해킹을 유발해서 (AI 훈련 메커니즘)
- 제텔카스텐에서 concept을 빠르게 쓰는 이유: 팩트 추출이라서 (인간 지식관리 프로세스)
- 메커니즘이 다르다. "빠르게"라는 표면적 비유만 같다.

**원칙 3 (서술) 위반**: "의미 도출에서만 깊은 사고가 필요하다"는 논문에서 나온 사실이 아니라 대화 중 추측이었는데, 사실처럼 서술했다. 논문은 "팩트 기반 QA에서 Fast가 낫다"만 보였지 "분석 단계에서는 Slow가 필요하다"는 검증하지 않았다.

## 탈락 판정

원본 노트의 도출 구조:

```
fact 1: Search-R1++에서 Fast > Slow (AI 에이전트)
fact 2: 다중 홉의 각 홉은 사실 조회 (정보검색)
→ 결론: 사고 깊이를 단계별로 조절하라 (일반 원리)
```

문제: fact 1→결론 사이에 "AI 에이전트 실험 → 일반 인지 전략"이라는 점프가 있고, "의미 도출에서는 깊게"는 근거 없는 추가.

## Observations

- [fact] "캐시는 어디에나 있다" 실패와 동일한 패턴 — 표면적 비유를 메커니즘으로 착각 #반복실패
- [fact] AI 훈련 메커니즘(보상 해킹)과 인간 지식관리 프로세스(팩트 추출)는 "빠르게"라는 비유만 공유하고 원인이 다르다 #비유공유
- [warning] 대화 중 자연스럽게 나온 연결이 반드시 유효한 메커니즘 공유는 아니다 — 대화의 흐름과 노트의 엄밀함은 기준이 다르다 #대화vs노트
