# Flow 프로젝트 폴더 생성 가이드

> AI Agent가 Flow 프로젝트 폴더를 자동 생성할 때 참고하는 문서

---

## 1. 작업 개요

### 목적
사용자의 작업 내역을 Flow.team에 등록할 수 있는 폴더 구조로 변환

### 입력 → 출력
```
입력: 작업 맥락 (DB 기록, 기존 파일, 사용자 설명)
출력: Flow 폴더 구조 + content 파일들
```

---

## 2. 맥락 수집 방법

### 2.1 orchestration_log DB 조회 (필수)

```bash
sqlite3 "C:\claude-workspace\_system\orchestration.db" "SELECT task_id, status, intent, context_summary, output_summary FROM orchestration_log WHERE intent LIKE '%키워드%' ORDER BY created_at ASC"
```

**핵심 필드:**
| 필드 | 용도 |
|------|------|
| `intent` | 작업 목적 → Task 제목에 활용 |
| `context_summary` | 배경/맥락 → content 파일에 활용 |
| `output_summary` | 산출물/결과 → content 파일에 활용 |
| `created_at` | 시간순 정렬 → Task 순서 결정 |

**시간순 정렬 중요**: `ORDER BY created_at ASC`로 작업 흐름 파악

### 2.2 기존 파일/폴더 탐색 (선택)

작업 산출물이 있는 폴더 구조 파악:
```bash
# 폴더 구조 확인
find "경로" -type d | head -30

# 파일 목록 확인
find "경로" -type f -name "*.py" -o -name "*.md" -o -name "*.json"
```

**Explore agent 활용** (토큰 효율):
```
Task tool → subagent_type: Explore → model: haiku
```

### 2.3 사용자 질문 (필요시)

불명확한 부분은 반드시 확인:
- 팀/업무영역 (프로젝트 제목에 필요)
- 프로젝트 목적
- 책임자

## 3. Flow 폴더 구조 규칙

### 3.1 계층 구조

```
프로젝트 (루트 폴더)
├── 그룹 (1depth) - Section
│   ├── Task (2depth) - 업무
│   │   ├── Task.md - content 파일
│   │   └── Subtask (3depth) - 하위업무
│   │       └── Subtask.md
│   └── Task (2depth)
└── 그룹 (1depth)
```

### 3.2 명명 규칙

| 항목 | 형식 | 예시 |
|------|------|------|
| 프로젝트 | `[팀_업무영역] 프로젝트명` | `[전략기획_데이터렌더링] SNS 기초 데이터 수집 자동화` |
| 그룹 | `숫자. 그룹명` | `0. 문제 정의 및 접근 방식 탐색` |
| Task | `숫자. 업무명` | `1. 현상파악` |
| Subtask | `숫자. 하위업무명` | `1. 계획수립` |
| Content | `폴더명.md` | `1. 현상파악.md` |

### 3.3 금지 문자

폴더명에 사용 불가:
```
/ \ : * ? " < > |
```

**대체 규칙**: `/` → `_` (예: `마케팅/BXC` → `마케팅_BXC`)

## 4. Task 분리 기준 (PDCA)

### 4.1 PDCA Cycle

| 단계 | 키워드 | 설명 |
|------|--------|------|
| **P**lan | 계획수립 | 방법/순서 계획 |
| **D**o | 실행/개발 | 실제 작업 수행 |
| **C**heck | 점검/확인 | 결과 검토, 한계점 확인 |
| **A**ct | 조치/채택 | 개선, 방향 전환 |

### 4.2 실패/전환 기록 방법

**중요**: 실패한 접근도 기록해야 함

```
0. 문제 정의 및 접근 방식 탐색/
├── 1. 현상파악/
├── 2. [실패한 접근] 시도/
│   ├── 1. 계획수립/     ← Plan
│   ├── 2. 실행/         ← Do
│   └── 3. 한계점 확인/   ← Check (왜 실패했는지)
└── 3. [새로운 접근] 채택/  ← Act (방향 전환)
```

### 4.3 Task 분리 조건

다음 경우 별도 Task로 분리:
- 결과물이 다른 경우
- 담당자가 다른 경우
- 마감시점에 시차가 있는 경우

## 5. Content 파일 작성 규칙

### 5.1 기본 구조

```markdown
## 개요
{이 Task가 무엇인지 1-2문장}

## 방법
1. 1단계
2. 2단계

## 산출물
- 파일명 또는 결과물
```

### 5.2 상황별 템플릿

**현상파악 (문제 정의)**
```markdown
## 배경
{현재 상황 설명}

## 문제점
- 문제1
- 문제2

## 목표
{해결하고자 하는 것}
```

**실패/한계점 확인**
```markdown
## 점검 결과 (Check)

### 발견된 문제
| 항목 | 문제점 |
|------|--------|
| A | 문제 설명 |

### 결론
{왜 이 방식이 안 되는지}
```

**방향 전환 (Act)**
```markdown
## 방향 전환 (Act)

### 새로운 아이디어
> "핵심 인사이트"

### 장점
1. 장점1
2. 장점2

### 결정
✅ {채택한 방식}
```

## 6. 작업 순서 (효율적)

### 6.1 권장 순서

```
1. 맥락 수집
   ├── DB 조회 (orchestration_log)
   ├── 기존 파일 탐색 (필요시)
   └── 사용자 확인 (불명확한 부분)

2. 구조 설계
   ├── 프로젝트 제목 확정
   ├── 그룹 목록 작성
   └── Task/Subtask 목록 작성

3. 사용자 확인
   └── 구조 제안 → 승인 받기

4. 폴더 생성
   └── mkdir -p로 한번에 생성

5. Content 파일 작성
   └── Write 도구로 각 파일 생성
```

### 6.2 병렬 처리

**병렬 가능**:
- DB 조회 + 파일 탐색
- 여러 content 파일 Write

**순차 필수**:
- 구조 확정 → 폴더 생성 → content 작성

## 7. 주의사항

### 7.1 대괄호 `[]` 처리

Bash에서 대괄호는 glob 패턴으로 인식됨

**해결**: 반드시 따옴표로 감싸기
```bash
# ✅ 올바른 방법
mkdir -p "[전략기획_데이터렌더링] 프로젝트명/0. 그룹명"

# ❌ 잘못된 방법 (glob 패턴으로 해석됨)
mkdir -p [전략기획_데이터렌더링] 프로젝트명/0. 그룹명
```

### 7.2 변수 사용 시

```bash
BASE="[전략기획_데이터렌더링] 프로젝트명"
mkdir -p "$BASE/0. 그룹명/1. Task명"
```

### 7.3 PowerShell 이스케이프 문제

복잡한 한글 경로는 PowerShell보다 **Bash (mkdir -p)** 권장

### 7.4 Content 파일명 = 폴더명

```
1. 현상파악/
└── 1. 현상파악.md   ← 폴더명과 동일해야 함
```

## 8. 프로젝트 설명 작성

### 형식
```
1. 목적: {프로젝트 수행 이유, 완수하면 얻는 것}
2. 기간: {시작일} ~
3. 책임: {팀명 직책 이름}
```

### 예시
```
1. 목적: X, Facebook, 네이버 블로그 채널의 기초 데이터를 자동 수집하여 휴먼에러를 최소화한다.
2. 기간: 2026.01.14 ~
3. 책임: 전략기획팀 정상필
```

## 9. 체크리스트

### 작업 전
- [ ] orchestration_log에서 관련 task 조회했는가?
- [ ] 시간순으로 정렬하여 흐름 파악했는가?
- [ ] 팀/업무영역 확인했는가?
- [ ] 프로젝트 제목 사용자 승인 받았는가?

### 구조 설계
- [ ] 0번 그룹에 배경/문제정의 포함했는가?
- [ ] 실패한 접근도 기록했는가? (PDCA)
- [ ] Task가 PDCA 또는 문제해결 단계로 분리되었는가?

### 폴더 생성
- [ ] 대괄호 경로를 따옴표로 감쌌는가?
- [ ] mkdir -p로 한번에 생성했는가?

### Content 작성
- [ ] 파일명이 폴더명과 동일한가?
- [ ] 각 파일에 개요/방법/산출물 포함했는가?

## 10. 예시: 전체 흐름

### 입력
```
사용자: "convert 폴더를 Flow 프로젝트로 만들어줘"
```

### 1단계: 맥락 수집
```sql
SELECT * FROM orchestration_log
WHERE intent LIKE '%convert%' OR intent LIKE '%extension%'
ORDER BY created_at ASC
```

### 2단계: 구조 제안
```
프로젝트: [전략기획_데이터렌더링] SNS 기초 데이터 수집 자동화

그룹:
0. 문제 정의 및 접근 방식 탐색
1. 데이터 수집 도구 개발
2. 데이터 구조 설계
...
```

### 3단계: 사용자 승인
```
"이 구조로 진행할까요?"
```

### 4단계: 폴더 생성
```bash
BASE="[전략기획_데이터렌더링] SNS 기초 데이터 수집 자동화"
mkdir -p "$BASE/0. 문제 정의 및 접근 방식 탐색/1. 현상파악"
...
```

### 5단계: Content 작성
```
Write 도구로 각 .md 파일 생성
```
