# 크리에이터 실시간 대시보드

SNS 6개 플랫폼 게시물 조회수를 병렬 수집하고 Go Gin 대시보드로 실시간 시각화하는 프로젝트. Metabase 대체 목표.

## 개요

collect-posts.js(Playwright)가 6개 플랫폼을 병렬 수집하여 로컬 SQLite에 저장하고, Go Gin 서버가 해당 DB를 읽기전용으로 열어 REST API + ECharts 대시보드를 제공한다. 수집 주기는 1시간, 대시보드 폴링은 1분.

## 코드 구성

**모듈**
- collect-posts: 6개 플랫폼 병렬 수집 + SQLite/webhook 이중 저장 (JavaScript, Playwright)
- sns-dashboard: Go Gin REST API + ECharts 대시보드 서버

**인프라**
- sns-dashboard.db: 로컬 SQLite (WAL 모드), 수집기가 쓰고 서버가 읽기
- n8n webhook: 기존 NAS DB(post_view_snapshots)로의 이중 저장 경로
