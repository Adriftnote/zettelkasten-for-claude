# 벡터 시맨틱 검색

basic-memory 지식베이스에 벡터 임베딩 기반 시맨틱 검색을 추가하는 프로젝트

## 📖 개요

basic-memory의 FTS5 키워드 검색을 보완하여, 의미 기반 유사도 검색을 제공합니다. "컨텍스트 관리"를 검색하면 "Four-Bucket Optimization" 같은 의미적으로 관련된 노트도 찾아줍니다. basic-memory 자체는 수정하지 않고, 별도 sqlite-vec DB + Python 스크립트로 구현.

## 코드 구성

**모듈**
- vecsearch: 벡터 시맨틱 검색 메인 스크립트 (인덱싱, 동기화, 검색)
- vec-watcher: memory.db 변경 감시 → 자동 sync (선택적)

**함수 (DB)**
- init-vector-db: sqlite-vec DB 초기화 및 스키마 생성
- get-entities-from-memory-db: memory.db에서 entity 목록 조회

**함수 (청킹)**
- strip-frontmatter: YAML 프론트매터 제거
- chunk-markdown: 마크다운 → ## 헤더 기준 청크 분할

**함수 (임베딩)**
- get-embedder: fastembed 모델 싱글턴 로더
- serialize-f32: float 벡터 → bytes 직렬화
- embed-texts: 텍스트 리스트 → 벡터 배열 변환

**함수 (파일 I/O)**
- read-entity-file: project 경로 기반 마크다운 파일 읽기

**함수 (인덱싱)**
- index-entity: entity 단위 파이프라인 (읽기→청킹→임베딩→저장)
- delete-entity-chunks: entity의 청크 및 벡터 삭제

**함수 (CLI 명령)**
- cmd-index: 전체 재인덱싱 명령
- cmd-sync: 증분 동기화 명령
- cmd-search: 시맨틱 검색 명령
- cmd-stats: 인덱스 통계 명령
- main-vecsearch: CLI 엔트리포인트 (argparse)

**함수 (Watcher)**
- run-sync-watcher: subprocess로 vecsearch sync 호출
- main-vec-watcher: watchfiles 루프 (memory.db 감시)
