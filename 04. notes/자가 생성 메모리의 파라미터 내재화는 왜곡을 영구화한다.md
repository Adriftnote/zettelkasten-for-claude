# 자가 생성 메모리의 파라미터 내재화는 왜곡을 영구화한다

EMPO²식 자기강화 학습에서, 왜곡된 자가 생성 팁이 파라미터에 증류되면 왜곡의 출처 추적·교정이 극히 어려워진다. 이는 Context Poisoning이 파라미터 레벨로 확장된 것이다.

## 도출 근거

다음 facts의 조합에서 도출됨:

1. **EMPO²는 자가 생성 팁을 파라미터에 내재화한다** — 메모리 증강 롤아웃(팁 있음)으로 높은 보상 궤적을 생성한 뒤, 팁을 제거하고 해당 궤적을 재현하도록 학습(오프-정책 증류). 결과적으로 팁의 지식이 모델 가중치에 영구 반영된다.
2. **Context Poisoning은 오류가 컨텍스트에 축적되면 자기강화된다** — 오류가 다시 컨텍스트로 들어가면 모델이 이를 사실로 인식하고, 더 큰 오류를 생성하는 피드백 루프가 형성된다. 누적될수록 심화되고, 원본 오류를 찾기 어려워진다.
3. **표면적 정렬: 보상 피드백이 내부 기준을 어떻게 바꿨는지 추적할 수 없다** — 출력만 관찰 가능하고, 보상 신호가 수십억 파라미터에 분산 반영되므로, 모델이 의도대로 학습했는지 아니면 겉으로만 맞추는 기준을 만들었는지 구분할 수 없다.

→ 따라서: **EMPO²의 자기강화 학습 루프에서 왜곡된 팁이 파라미터에 내재화되면, (1) Context Poisoning처럼 자기강화되지만 컨텍스트가 아닌 파라미터에 고착되어 세션 초기화로도 복구 불가능하고, (2) 표면적 정렬 문제로 인해 왜곡이 내재화됐는지 외부에서 감지할 수 없으며, (3) 모델이 자가 생성한 팁이 학습 데이터이므로 기존 데이터 감사(audit) 방법론이 적용되지 않는다.**

## 위험 메커니즘 분석

### 1. Context Poisoning의 파라미터 레벨 확장

기존 Context Poisoning은 세션 단위 문제다 — 컨텍스트를 초기화하면 해소된다. 그러나 EMPO²의 오프-정책 증류는 오염된 팁을 파라미터에 굳힌다. 이는 "세션을 넘어 지속되는 Context Poisoning"이다.

```
Context Poisoning:   오류 → 컨텍스트 축적 → 자기강화 (세션 한정, 초기화로 복구)
파라미터 내재화:     오류 팁 → 파라미터 증류 → 영구 고착 (초기화 불가)
```

### 2. 편향 세탁 (Bias Laundering)

기존 SFT/RLHF는 "어떤 데이터로 학습했는가"를 감사할 수 있다. EMPO²에서는 학습 데이터가 모델 자가 생성물이므로, 편향의 출처가 외부 데이터가 아닌 모델 자체가 된다. 이는 의도적 악용 시 책임 소재를 모호하게 만든다.

### 3. 보상 함수 단일 의존의 위험

EMPO²에서 사람이 직접 통제하는 지점은 보상 함수 하나뿐이다. 보상 함수가 모호한 영역(뉴스 추천, 의료 조언, 정치 정보 등)에서는, 모델이 "높은 보상을 받는 팁"을 자가 생성하고 내재화하는 과정에서 의도치 않은 편향이 체화될 수 있다.

### 4. 기존 RL 대비 증폭 효과

기존 GRPO/PPO는 보상 → 행동 강화의 1단계 피드백이다. EMPO²는 보상 → 행동 강화 → 팁 자가 생성 → 팁으로 더 강한 행동 → 파라미터 증류의 다단계 자기강화 루프다. 한 번 잘못된 방향이 잡히면 교정 비용이 기하급수적으로 증가한다.

## Observations

- [fact] EMPO²의 오프-정책 증류는 팁 조건부 궤적을 팁 없는 정책으로 재현하도록 학습하여, 메모리 지식을 파라미터에 영구 반영한다 #EMPO2 #증류
- [fact] Context Poisoning은 세션 초기화로 복구 가능하지만, 파라미터 내재화된 왜곡은 재훈련 없이 복구할 수 없다 #파라미터고착
- [fact] 표면적 정렬 문제로 인해 왜곡이 내재화됐는지 외부에서 감지하는 것은 원리적으로 제한된다 #감사불가
- [fact] 학습 데이터가 모델 자가 생성물이면, 기존 데이터 감사 방법론이 적용되지 않는다 #편향세탁
- [method] 보상 함수 설계 시 객관적 검증 가능한 영역으로 제한하면 위험을 줄일 수 있다 #완화방안
- [question] 팁 생성 단계에 외부 검증 루프를 삽입하면 자기강화 위험을 차단할 수 있는가? #후속질문
- [question] 내재화된 편향을 사후 탐지하는 mechanistic interpretability 기법이 가능한가? #후속질문
