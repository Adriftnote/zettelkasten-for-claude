# MoE (Mixture of Experts)

입력에 따라 전체 파라미터 중 일부 전문가(Expert)만 선택적으로 활성화하는 신경망 아키텍처

## 📖 개요

Mixture of Experts(MoE)는 트랜스포머의 FFN(Feed-Forward Network) 레이어를 여러 개의 전문가 서브네트워크로 분할하고, 라우터(Router)가 각 입력 토큰을 소수의 전문가에게만 배정하는 구조입니다. 전체 파라미터 수는 크지만 한 번의 추론에 일부만 활성화되므로, 큰 모델의 표현력과 작은 모델의 연산 비용을 동시에 달성합니다.

## 🎭 비유

종합병원과 같습니다. 의사가 100명(전체 파라미터) 있지만, 환자(입력 토큰)가 오면 접수처(라우터)에서 증상에 맞는 전문의 2~3명(활성 전문가)에게만 배정합니다. 모든 의사가 동시에 진료하지 않아도 병원 전체의 전문성은 유지됩니다.

## ✨ 특징

- **조건부 연산**: 입력마다 활성화되는 전문가가 다름 (Conditional Computation)
- **스케일링 효율**: 파라미터 수 대비 연산량(FLOPs)이 적음 — 예: 전체 2.3B, 활성 1.5B
- **자연 특화**: 학습 과정에서 전문가가 특정 도메인/모달리티에 알아서 특화됨
- **라우팅**: 일반적으로 Top-K 게이팅 — 입력 토큰별로 점수가 높은 K개 전문가만 선택

## 💡 예시

### 구조

```
입력 토큰
    ↓
[라우터: 각 전문가 점수 계산]
    ├── Expert 1 (점수 0.7) ✅ 활성
    ├── Expert 2 (점수 0.5) ✅ 활성 (Top-2)
    ├── Expert 3 (점수 0.2) ❌ 비활성
    ├── ...
    └── Expert 16 (점수 0.1) ❌ 비활성
    ↓
활성 전문가 출력의 가중합 → 최종 출력
```

### Dense vs MoE 비교

| | Dense | MoE |
|--|-------|-----|
| 추론 시 활성 파라미터 | 전부 | 일부 (예: 전체의 40~65%) |
| 전문가 수 | 1개 (공유 FFN) | N개 (예: 8~1008개) |
| 메모리 | 파라미터 수에 비례 | Dense보다 큼 (전체 Expert 저장) |
| 연산량 | 파라미터 수에 비례 | Dense보다 작음 (일부만 활성) |

### 멀티모달에서의 전문가 특화 (Beyond Language Modeling 논문)

학습 후 관찰된 패턴:
- 텍스트 토큰 → 대부분 특정 전문가 집합으로 라우팅 (텍스트 특화)
- 비전 토큰 → 범용 전문가 활용, 이해+생성이 동일 전문가 공유
- 이 특화는 사전 지정이 아니라 학습 과정에서 자연 형성됨

## ⚠️ 한계

- **로드 밸런싱**: 특정 전문가에 토큰이 몰리는 문제 → 보조 손실(auxiliary loss)로 완화
- **메모리 비용**: 전체 전문가를 메모리에 올려야 함 → 추론 시 모델 병렬화 필요
- **학습 불안정**: 라우터 학습이 불안정할 수 있음
