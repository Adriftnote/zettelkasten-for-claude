# Using Variables in Card Titles

## 📖 개요

Metabase에서 대시보드의 필터값이나 동적 데이터를 카드의 제목에 반영하여, 제목이 현재 표시 중인 데이터를 정확히 나타내도록 하는 패턴입니다. 예를 들어 "서울 지역의 판매" 필터를 선택하면 제목도 자동으로 "서울 지역의 판매"로 변경됩니다.

## 🎭 비유

신문의 헤드라인 같은 것입니다. 신문사가 "경제 뉴스" 섹션을 클릭하면 헤드라인이 경제 관련 기사로 변경되고, "스포츠" 섹션을 클릭하면 스포츠 기사로 변경됩니다. 섹션이 자동으로 제목을 결정합니다.

## ✨ 특징

- **Dynamic Titles**: 필터 선택에 따라 제목 자동 변경
- **Context Clarity**: 현재 보는 데이터가 무엇인지 명확
- **Professional Look**: 정적 제목보다 세련됨
- **Easy to Understand**: 사용자가 현재 필터 상황을 한눈에 파악
- **Automatic Sync**: 필터 변경 시 제목도 자동 업데이트

## 💡 예시

### 예시 1: 기본 변수 사용

```markdown
## 변수 사용 패턴

### 상황: 지역별 판매 대시보드

필터명: region (지역)
옵션: 서울, 부산, 대구

#### 정적 제목 (나쁜 예)
```
Title: "월간 판매액"
→ 문제: 어느 지역인지 불명확
```

#### 동적 제목 (좋은 예)
```
Title: "{{region}} 지역의 월간 판매액"

필터값 변경에 따른 제목 변화:
- 서울 선택 → "서울 지역의 월간 판매액"
- 부산 선택 → "부산 지역의 월간 판매액"
- 대구 선택 → "대구 지역의 월간 판매액"
```

### 예시 2: 여러 변수 조합

```markdown
## 복합 변수 사용

### 필터 구성
- 연도: year (2022, 2023, 2024)
- 분기: quarter (Q1, Q2, Q3, Q4)
- 부서: department (영업, 마케팅, 운영)

### 동적 제목 예시
```
"{{year}}년 {{quarter}} {{department}}부서 매출"

필터 선택:
- 2024년, Q1, 영업 선택
  → "2024년 Q1 영업부서 매출"

- 2023년, Q4, 마케팅 선택
  → "2023년 Q4 마케팅부서 매출"
```

### 예시 3: 조건부 텍스트 포함

```markdown
## 동적 제목에 컨텍스트 추가

### 패턴 1: 시간 기반 텍스트
```
"{{year}}-{{month}} {{region}} 판매 현황
{{#if current_month}}
(최신 데이터)
{{/if}}"
```

### 패턴 2: 값에 따른 이모지 추가
```
" {{region}} 판매액: ${{sales}}
{{#if sales > target}}✓ 목표 달성{{else}}⚠️ 목표 미달{{/if}}"
```

### 패턴 3: 다국어 지원
```
"{{region}}: {{year}}-{{month}} Sales Report
({{locale}}) 보고서"

- 한글 선택: "서울: 2024-01 Sales Report (KO) 보고서"
- 영문 선택: "Seoul: 2024-01 Sales Report (EN) Report"
```

### 예시 4: 카드별 변수 적용 예시

```markdown
## 대시보드 레이아웃: 월별 판매 분석

┌──────────────────────────────┐
│ {{year}}-{{month}} {{region}} 판매 현황   │ ← 변수 포함 제목
├──────────────────────────────┤
│                              │
│    [판매액 차트]              │
│                              │
├──────────────────────────────┤
│ {{region}}의 상품별 매출 TOP 10 │ ← 변수 포함 제목
├──────────────────────────────┤
│ [테이블: 상품별 매출]         │
│                              │
├──────────────────────────────┤
│ {{region}} 시간대별 매출      │ ← 변수 포함 제목
│ ({{period}} 기준)             │
├──────────────────────────────┤
│ [시간대별 판매 그래프]        │
│                              │
└──────────────────────────────┘

각 카드의 제목:
- 필터 변경 → 모든 제목 자동 업데이트
- 사용자는 현재 보는 데이터가 무엇인지 명확하게 파악
```

### 예시 5: 변수 구현 단계별 가이드

```markdown
## 변수를 사용한 동적 제목 적용

### 1단계: 대시보드 필터 생성
- 대시보드 편집 모드 진입
- "필터 추가" 클릭
- 필터명: region (영문 권장)
- 표시명: "지역" (한글)
- 옵션: 서울, 부산, 대구, ...

### 2단계: 카드 제목 설정
- 카드 우측 상단 "설정" 아이콘 클릭
- "카드 제목 및 설명" 메뉴
- 제목 필드 클릭

### 3단계: 변수 문법 입력
제목 필드에:
```
"{{region}} 지역의 판매액"
```

또는 여러 변수:
```
"{{year}}-{{month}} {{region}} 분석"
```

### 4단계: 변수 할당
- 변수명 {{region}}을 필터와 연결
- "필터와 연결" 옵션 선택
- 대시보드의 "region" 필터와 매핑

### 5단계: 테스트
- 편집 모드 나가기 (저장)
- 필터 드롭다운에서 값 선택
- 제목이 자동으로 변경되는지 확인

### 6단계: 저장
- 모든 카드 확인
- "저장" 클릭
```

### 예시 6: 변수 사용 체크리스트

```markdown
## 동적 제목 구현 체크리스트

### 계획 단계
□ 대시보드에 사용할 필터 목록 작성
□ 각 필터의 변수명 결정 (영문, snake_case)
□ 제목 템플릿 작성

### 구현 단계
□ 필터 생성 (필터명 = 변수명)
□ 각 카드의 제목에 {{variable}} 형식으로 입력
□ 필터와 변수 매핑 확인
□ 필터와 카드의 연결 확인

### 검증 단계
□ 필터값 변경 시 제목 변경 여부 확인
□ 모든 필터값에 대해 테스트
□ 제목이 정확하게 표시되는지 확인
□ 특수 문자나 공백 처리 확인

### 배포 단계
□ 모든 카드의 동적 제목 확인
□ 사용자 교육 자료 준비
□ 변수명과 필터명 문서화
□ 저장 및 공개
```

## 🛠️ 해결/적용 방법

### 1단계: 변수 식별
```
대시보드에 사용할 필터들:
- region (지역)
- year (연도)
- product (상품)

각각에 대해 변수명 정의
```

### 2단계: 제목 템플릿 작성
```
기본 형식:
"{{변수명}} 설명"

예:
"{{region}} 지역의 {{year}}년 판매"
```

### 3단계: Metabase에서 구현
- 필터 생성
- 각 카드의 제목을 템플릿으로 변경
- 변수와 필터 연결

### 4단계: 테스트
```
테스트 케이스:
1. 필터값 선택 → 제목 확인
2. 필터값 변경 → 제목 업데이트 확인
3. 다중 필터 → 모든 변수가 포함되는지 확인
```

### 5단계: 문서화
- 변수명과 필터명의 대응 관계 기록
- 템플릿 예시 저장
