# Turing Machine (튜링 머신)

**"모든 계산 가능한 문제를 풀 수 있는 이론적 컴퓨터 모델"**

Alan Turing이 1936년 논문에서 제안. 현대 컴퓨터의 이론적 기초이자, "계산 가능성"의 수학적 정의.

## 📖 개요

핵심 질문: **"기계로 계산할 수 있는 것의 한계는 무엇인가?"**

```
[무한 테이프] ← → [읽기/쓰기 헤드] ← → [상태 기계]
   0 1 0 1 1 0 ...         ↑              규칙 테이블
                      현재 위치
```

## 🎭 비유

### 끝없는 종이 테이프와 펜

```
[튜링 머신 = 극도로 단순한 사무원]

도구:
- 무한히 긴 종이 테이프 (칸마다 기호 하나)
- 연필과 지우개
- 규칙표 (어떤 상황에서 뭘 할지)

작업:
1. 현재 칸의 기호를 읽는다
2. 규칙표를 보고 행동 결정:
   - 기호 쓰기/지우기
   - 왼쪽/오른쪽으로 한 칸 이동
   - 다음 상태로 전환
3. 반복 (멈춤 명령까지)
```

## 💡 구성 요소

| 구성 요소 | 설명 | 현대 컴퓨터 대응 |
|----------|------|-----------------|
| **테이프** | 무한한 1차원 메모리 | RAM/저장장치 |
| **헤드** | 읽기/쓰기 장치 | CPU의 메모리 접근 |
| **상태** | 현재 기계의 "기분" | 프로그램 카운터 |
| **전이 함수** | (상태, 기호) → 행동 | 프로그램/명령어 |

## 💻 동작 예시

### 이진수 1 더하기

```
초기: ... 0 1 0 1 1 □ □ ...  (□ = 빈칸)
              ↑ 헤드 (오른쪽 끝에서 시작)

규칙:
- 1을 만나면 → 0으로 바꾸고 왼쪽으로
- 0을 만나면 → 1로 바꾸고 멈춤
- 빈칸이면 → 1 쓰고 멈춤

실행:
1. 1 읽음 → 0 쓰고 왼쪽: ... 0 1 0 1 0 □ ...
2. 1 읽음 → 0 쓰고 왼쪽: ... 0 1 0 0 0 □ ...
3. 0 읽음 → 1 쓰고 멈춤: ... 0 1 1 0 0 □ ...

결과: 10111 → 11000 (23 + 1 = 24) ✓
```

## 📊 튜링 머신의 의미

### Church-Turing 논제

> "효과적으로 계산 가능한 모든 것은 튜링 머신으로 계산할 수 있다"

| 의미 | 설명 |
|------|------|
| **계산의 정의** | "계산 가능" = "튜링 머신으로 풀 수 있음" |
| **컴퓨터의 한계** | 튜링 머신이 못 푸는 건 어떤 컴퓨터도 못 품 |
| **보편성** | 모든 프로그래밍 언어는 튜링 머신과 동치 |

### 결정 불가능 문제

튜링이 증명한 것: **"풀 수 없는 문제가 존재한다"**

```
[정지 문제 (Halting Problem)]

질문: 프로그램 P가 입력 X에 대해 멈출까, 무한 루프일까?

답: 일반적으로 판별 불가능!

증명: 귀류법 (자기 참조 역설)
```

## 🔧 변형들

| 변형 | 설명 | 계산 능력 |
|------|------|----------|
| **다중 테이프** | 테이프 여러 개 | 동일 (더 빠름) |
| **비결정적** | 여러 선택지 동시 탐색 | 동일 |
| **양자 튜링 머신** | 양자 상태 활용 | 동일 (더 빠름) |

## ⚠️ 흔한 오해

| 오해 | 사실 |
|------|------|
| "실제 컴퓨터다" | 이론적 모델, 물리적 구현 X |
| "무한 메모리 필요" | 이론상 무한, 실제론 유한으로 근사 |
| "느리다" | 효율성이 아닌 계산 가능성에 관한 모델 |

## 💡 현대적 의의

```

Python, JavaScript, C, Java...
↓
모두 튜링 머신과 동일한 계산 능력
↓
하나로 풀 수 있으면 다른 것으로도 풀 수 있음
```
