# Cache (캐시)

**자주 쓰는 데이터를 가까이 저장**해두는 임시 저장소입니다.

## 📖 개요

캐시는 "비싼 연산의 결과"나 "멀리서 가져와야 하는 데이터"를 가까운 곳에 저장해두고 재사용하는 기법입니다. 속도와 메모리 사이의 트레이드오프입니다.

## 🎭 비유

| 상황 | 캐시 없음 | 캐시 있음 |
|------|----------|----------|
| **공부** | 매번 도서관 가서 책 찾기 | 자주 보는 책은 책상 위에 |
| **요리** | 재료 필요할 때마다 마트 | 냉장고에 미리 사둠 |
| **웹** | 매번 서버에서 다운로드 | 방문했던 이미지는 PC에 저장 |

## ✨ 핵심 원리

| 원리 | 설명 |
|------|------|
| **지역성 (Locality)** | 최근 쓴 것을 또 쓸 확률 높음 |
| **트레이드오프** | 속도 ↑ = 메모리 사용 ↑ |
| **적중률 (Hit Rate)** | 캐시에서 찾은 비율 (높을수록 좋음) |

## 💡 캐시의 종류

| 종류 | 저장 위치 | 용도 |
|------|----------|------|
| **CPU 캐시** | CPU 칩 내부 (L1/L2/L3) | 명령어, 데이터 |
| **메모리 캐시** | RAM | DB 쿼리 결과, 세션 |
| **디스크 캐시** | SSD/HDD | 파일, 웹페이지 |
| **브라우저 캐시** | PC 로컬 | 이미지, CSS, JS |
| **KV-Cache** | [[VRAM]] | LLM의 [[Attention]] 계산값 |

## ⚠️ 캐시의 문제

| 문제 | 설명 |
|------|------|
| **캐시 미스** | 캐시에 없어서 원본에서 가져와야 함 |
| **캐시 무효화** | 원본이 바뀌면 캐시도 갱신해야 함 |
| **메모리 부족** | 캐시가 너무 커지면 관리 필요 |

> "컴퓨터 과학에서 어려운 것은 딱 두 가지: 캐시 무효화와 이름 짓기"
> — Phil Karlton
