# Attention (어텐션)

**"어디를 봐야 하는지"** 결정하는 메커니즘입니다. 트랜스포머 모델의 핵심 구성요소입니다.

## 📖 개요

어텐션은 입력의 각 부분이 다른 부분과 얼마나 관련 있는지 계산합니다. 이를 통해 모델이 중요한 정보에 집중할 수 있습니다.

## 🎭 비유

**독해 시험 비유**:
```
문제: "철수가 뭘 먹었나요?"

지문: "철수는 학교에 갔다. 영희는 사과를 샀다. 철수는 빵을 먹었다."

어텐션:
  "철수는 학교에 갔다"  →  ██░░░░  (철수 언급, 먹는 거 아님)
  "영희는 사과를 샀다"  →  ░░░░░░  (무관)
  "철수는 빵을 먹었다"  →  ██████  (정답!)
```

## ✨ Query, Key, Value

어텐션의 세 가지 핵심 요소:

| 요소 | 역할 | 비유 |
|------|------|------|
| **Query (Q)** | 내가 찾고 싶은 것 | 검색어 |
| **Key (K)** | 각 항목의 라벨/제목 | 책 제목 |
| **Value (V)** | 실제 정보/내용 | 책 내용 |

```
도서관 비유:
Q: "고양이 키우기" (내가 찾는 주제)
K: 각 책의 제목 (비교 대상)
V: 책의 실제 내용 (가져올 정보)

→ Q와 K를 비교해서 관련 있는 책의 V를 가져옴
```

## 💡 계산 과정

```
1. Q와 모든 K의 유사도 계산 (내적)
2. Softmax로 가중치 변환 (합=1)
3. 가중치로 V를 가중 합산

수식: Attention(Q,K,V) = softmax(QK^T / √d) × V
```

**예시**:
```
입력: "고양이가 생선을 먹었다"
"먹었다"의 어텐션 계산:

           Q("먹었다") · K(각 단어)
단어        유사도    가중치    V 기여도
─────────────────────────────────────
고양이가     0.8      0.3      ███░░░
생선을       1.2      0.5      █████░
먹었다       0.5      0.2      ██░░░░

→ "먹었다"는 "생선을"에 가장 많이 주목
```

## 🔀 Self-Attention vs Cross-Attention

| 종류 | Q, K, V 출처 | 용도 |
|------|-------------|------|
| **Self-Attention** | 모두 같은 시퀀스 | 문장 내 관계 파악 |
| **Cross-Attention** | Q≠K,V (다른 시퀀스) | 번역, 이미지→텍스트 |

## ⚠️ 어텐션의 비용

| 문제 | 원인 | 해결책 |
|------|------|--------|
| O(n²) 복잡도 | 모든 토큰 쌍 비교 | Sparse Attention |
| [[VRAM]] 사용량 | K, V 저장 필요 | [[kv-cache-optimization\|KV-Cache]] |
| 긴 컨텍스트 한계 | 메모리 폭발 | 슬라이딩 윈도우 |
