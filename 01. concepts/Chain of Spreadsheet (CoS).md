# Chain of Spreadsheet (CoS)

스프레드시트 질의를 "테이블 찾기 → 답 구하기" 2단계로 나눠 AI 정확도를 높이는 추론 프레임워크

## 📖 개요

하나의 시트에 여러 표가 섞여 있을 때, AI에게 "매출 합계가 얼마야?"라고 바로 물으면 엉뚱한 표의 숫자를 참조할 수 있다. CoS는 Chain of Thought(단계적 사고)에서 영감받아, 질문에 답하기 전에 먼저 "어떤 표를 봐야 하는지"를 식별하는 단계를 추가한다. 사람이 복잡한 엑셀을 볼 때도 "이 시트에서 내가 볼 표가 어디지?" → "그 표에서 답을 찾자"로 2단계를 거치는 것과 같다.

## 🎭 비유

도서관에서 질문에 답하는 두 가지 방법:
- **기존 방식 (1단계)**: "한국 GDP가 얼마야?" → 도서관 전체 책을 다 펼쳐놓고 한꺼번에 답을 찾으려 함 → 헷갈림
- **CoS (2단계)**: "한국 GDP가 얼마야?" → ① 먼저 경제 섹션의 통계 연감을 찾고 → ② 그 책에서 GDP 항목을 찾음 → 정확

## ✨ 2단계 구조

### Stage 1: 테이블 식별 (Table Identification)

```
입력: 압축된 시트 전체 + 사용자 질문
출력: 관련 테이블의 위치 (셀 범위)

프롬프트 구조:
"이 스프레드시트에서 [질문]에 답하려면 어떤 테이블을 봐야 하는가?
해당 테이블의 셀 범위를 알려줘."

예시 응답: "A1:D15의 매출 테이블이 관련 있음"
```

이 단계에서는 **답을 구하지 않는다**. 오직 "어디를 볼지"만 결정한다.

### Stage 2: 응답 생성 (Answer Generation)

```
입력: Stage 1에서 식별된 테이블 + 사용자 질문
출력: 최종 답변

프롬프트 구조:
"아래 테이블을 참고하여 [질문]에 답하시오.
[Stage 1에서 추출한 테이블 데이터]"
```

범위가 좁혀졌으므로 AI가 집중해서 정확한 답을 생성한다.

## 💡 왜 나눠야 효과적인가

```
한 시트에 3개 표가 있는 경우:

┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│ 매출 테이블  │  │ 직원 테이블  │  │ 예산 테이블  │
│ A1:D15      │  │ F1:H20      │  │ A25:E35     │
└─────────────┘  └─────────────┘  └─────────────┘

질문: "Q4 매출 합계는?"

1단계 방식: 3개 표 데이터를 한꺼번에 보고 답 시도
→ 직원 테이블의 숫자를 매출로 착각할 위험

CoS 방식:
  Stage 1: "매출 테이블(A1:D15)을 봐야 함" ← 범위 특정
  Stage 2: A1:D15만 보고 Q4 합계 계산 ← 정확한 답
```

핵심은 **주의 범위를 좁히는 것**이다. LLM의 컨텍스트에 무관한 데이터가 적을수록 정확도가 올라간다.

## 📊 효과

| 비교 | Accuracy |
|------|----------|
| GPT-4 + 압축 (1단계) | 약 60% |
| GPT-4 + 압축 + CoS (2단계) | 74.3% |

2단계로 나누는 것만으로 약 14%p 정확도 향상.

## ⚡ 직접 활용하는 프롬프트 패턴

CoS를 직접 구현하지 않더라도, 프롬프트를 2번 나눠 보내면 같은 효과를 낼 수 있다:

```
# 1번째 프롬프트 (테이블 식별)
"이 엑셀 캡처를 보고, [질문]에 답하려면 어떤 영역을 참고해야 하는지 알려줘.
셀 범위와 해당 테이블의 용도를 설명해줘."

# 2번째 프롬프트 (답변 생성)
"[1번 답변에서 특정된 영역]의 CSV 데이터야. 
이 데이터로 [질문]에 답해줘."
```

또는 하나의 프롬프트에서 단계를 명시:

```
"다음 데이터에서 [질문]에 답해줘.
Step 1: 먼저 관련된 테이블 영역을 식별하고
Step 2: 해당 영역의 데이터만으로 답을 구해줘"
```
