# Von Neumann Architecture (폰 노이만 구조)

**"프로그램과 데이터를 같은 메모리에 저장하는 컴퓨터 구조"**

John von Neumann이 1945년 EDVAC 보고서에서 제안. 현대 거의 모든 컴퓨터의 기본 설계.

## 📖 개요

핵심 아이디어: **"저장된 프로그램 (Stored Program)"**

```
┌─────────────────────────────────────┐
│         메모리 (Memory)              │
│  ┌──────────┬──────────┐            │
│  │ 프로그램 │  데이터  │            │
│  └──────────┴──────────┘            │
└─────────────┬───────────────────────┘
              ↕ (버스)
┌─────────────┴───────────────────────┐
│     CPU (중앙처리장치)               │
│  ┌──────────┬──────────┐            │
│  │ 제어장치 │ 연산장치 │            │
│  └──────────┴──────────┘            │
└─────────────────────────────────────┘
      ↕                    ↕
  [입력장치]          [출력장치]
```

## 🎭 비유

### 이전 vs 이후

```
[이전: 하드와이어드 컴퓨터]
프로그램 = 물리적 배선
바꾸려면 → 배선 다시 연결 (며칠~몇 주)

[이후: 폰 노이만 구조]
프로그램 = 메모리에 저장된 숫자
바꾸려면 → 새 프로그램 로드 (몇 초)

비유: 전화기 vs 스마트폰
- 전화기: 하드웨어가 기능 결정
- 스마트폰: 앱(소프트웨어)이 기능 결정
```

## 💡 5대 구성 요소

| 구성 요소 | 역할 | 현대 예시 |
|----------|------|----------|
| **메모리** | 프로그램 + 데이터 저장 | RAM |
| **연산장치 (ALU)** | 산술/논리 연산 | CPU 내부 |
| **제어장치** | 명령어 해석, 실행 순서 제어 | CPU 내부 |
| **입력장치** | 외부 → 컴퓨터 | 키보드, 마우스 |
| **출력장치** | 컴퓨터 → 외부 | 모니터, 프린터 |

## 💻 명령어 실행 사이클

```
[Fetch-Decode-Execute 사이클]

1. Fetch (인출)
   PC(프로그램 카운터)가 가리키는 주소에서 명령어 읽기

2. Decode (해석)
   명령어가 뭘 하라는 건지 해석

3. Execute (실행)
   연산 수행, 결과 저장

4. PC 증가 → 다음 명령어로

반복...
```

### 예시

```
메모리:
[100]: ADD R1, R2    ← PC가 여기 가리킴
[101]: STORE R1, 200
[102]: ...

실행:
1. [100] 읽기 → ADD R1, R2
2. "R1과 R2를 더해서 R1에 저장"
3. 덧셈 실행
4. PC → 101
```

## 📊 장점과 한계

### 장점

| 장점 | 설명 |
|------|------|
| **유연성** | 프로그램만 바꾸면 다른 작업 가능 |
| **단순성** | 하나의 메모리, 하나의 버스 |
| **범용성** | 어떤 계산이든 가능 |

### 폰 노이만 병목 (Von Neumann Bottleneck)

```
[병목 현상]

CPU ←──── 버스 ────→ 메모리
     (좁은 통로)

문제: CPU는 빠른데 메모리 접근이 느림
     → CPU가 데이터 기다리며 놀고 있음

해결책:
- 캐시 (Cache) 추가
- 파이프라이닝
- 하버드 구조 (명령어/데이터 분리)
```

## 🔧 변형과 발전

| 구조 | 특징 | 용도 |
|------|------|------|
| **하버드 구조** | 명령어/데이터 메모리 분리 | 임베디드, DSP |
| **수정된 하버드** | 캐시는 분리, 메인 메모리는 통합 | 현대 CPU |
| **GPU 구조** | 많은 코어, 병렬 처리 | 그래픽, AI |

## ⚠️ 흔한 오해

| 오해 | 사실 |
|------|------|
| "폰 노이만이 발명" | 여러 사람의 아이디어, 폰 노이만이 정리 |
| "오래된 구조" | 80년이 지났지만 여전히 기본 |
| "비효율적" | 캐시 등으로 병목 완화, 여전히 가장 실용적 |
