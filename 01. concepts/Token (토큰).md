# Token (토큰)

AI 모델이 입력을 처리하는 최소 단위. 텍스트에서는 서브워드 조각, 이미지에서는 패치 벡터.

## 📖 개요

토큰은 AI 모델(특히 트랜스포머)이 데이터를 처리할 수 있는 형태로 변환한 최소 단위입니다. 텍스트는 토크나이저가 단어를 서브워드로 분리하고, 이미지는 인코더(ViT, RAE 등)가 패치 단위로 분할합니다. 최종적으로 토큰은 고정 길이의 실수 벡터(float 배열)가 되며, 이 형식이 모달리티에 관계없이 동일하기 때문에 텍스트·이미지·비디오를 하나의 모델로 처리하는 멀티모달 아키텍처가 가능합니다.

## 🎭 비유

레고 블록과 같습니다. 어떤 구조물(문장, 이미지)이든 표준 규격의 블록(토큰)으로 분해하면, 같은 조립 방식(트랜스포머)으로 처리할 수 있습니다.

## ✨ 특징

- **이산 ID → 연속 벡터**: 토크나이저가 문자열을 정수 ID로, 임베딩 레이어가 ID를 고정 차원 벡터로 변환
- **고정 차원**: 모달리티와 무관하게 동일한 차원의 벡터 (예: 4096차원)
- **서브워드 단위**: 텍스트 토큰은 단어보다 작은 단위 — "playing" → "play" + "ing"
- **시퀀스 처리**: 트랜스포머는 토큰의 시퀀스를 입력받아 다음 토큰을 예측

## 💡 예시

### 텍스트 토큰화 과정

```
"고양이가 귀엽다"

1. 토크나이저 (BPE 등): 서브워드 분리
   → ["고양이", "가", " 귀", "엽다"]

2. 어휘 사전 조회: 정수 ID 부여
   → [8274, 102, 5531, 9847]

3. 임베딩 레이어: ID → 실수 벡터
   → [
       [0.12, -0.55, 0.03, ..., 0.77],   ← 4096개의 float
       [0.87,  0.21, -0.44, ..., 0.33],
       ...
     ]
```

### 이미지 토큰화 과정

```
이미지 (256×256 픽셀)
= 256 × 256 × 3(RGB) = 196,608개의 숫자 (저수준, 중복 많음)

인코더 (ViT, RAE 등) 통과:
→ 256개의 토큰 × 각 1024차원 벡터 (고수준, 의미 압축)

각 토큰 = [0.82, -0.34, 0.91, ...]  ← 텍스트 토큰과 동일한 형식
```

### 멀티모달 통합이 가능한 이유

```
텍스트 토큰: [float × 4096] 의 시퀀스
이미지 토큰: [float × 4096] 의 시퀀스   ← 형식이 동일!
비디오 토큰: [float × 4096] 의 시퀀스

→ 이어붙이면 하나의 시퀀스
→ 트랜스포머 입장에서는 전부 "숫자 벡터의 나열"
```

### 데이터 형식 정리

| 단계 | 형식 | 예시 |
|------|------|------|
| 원본 (텍스트) | 문자열 | "고양이" |
| 토크나이저 출력 | 정수 ID | 8274 |
| 임베딩 출력 | float 벡터 (d차원) | [0.12, -0.55, ..., 0.77] |
| 트랜스포머 내부 | float 벡터 (d차원) | 문맥 반영된 벡터 |

## ⚠️ 주의

- 토큰 ≠ 단어. 한국어는 한 단어가 여러 토큰으로 쪼개지는 경우가 많음
- 토큰 수 = 비용. LLM API는 토큰 수 기준으로 과금
- 컨텍스트 윈도우: 모델이 한 번에 처리할 수 있는 최대 토큰 수 제한 있음
