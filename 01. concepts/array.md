# 배열 (Array)

배열은 동일한 타입의 여러 값을 메모리의 연속된 공간에 저장하는 선형 자료구조로, 인덱스를 통해 O(1)의 시간에 원소에 접근할 수 있습니다.

## 비유

배열은 영화관의 좌석처럼 생각할 수 있습니다. 각 좌석(원소)은 순서대로 번호가 매겨져 있고(인덱스), 좌석 번호를 알면 바로 그 위치로 갈 수 있습니다(O(1) 접근). 하지만 좌석을 추가하거나 제거하려면 전체 배열을 재정렬해야 합니다(삽입/삭제 비용).

## 메모리 구조

```
배열 선언: let arr = [1, 2, 3, 4, 5];

메모리 주소:  0x1000  0x1004  0x1008  0x100C  0x1010
메모리 값:    [1]     [2]     [3]     [4]     [5]
인덱스:       0       1       2       3       4

주소 계산: arr[i] = 시작_주소 + (인덱스 × 원소_크기)
예) arr[2]의 주소 = 0x1000 + (2 × 4바이트) = 0x1008
```

## 주요 특성

- **메모리 배치**: 연속된 메모리 공간에 선형으로 배치
- **고정 크기**: 생성 시 크기가 정해지면 변경 불가 (대부분의 언어)
- **인덱스 기반**: 0부터 시작하는 정수 인덱스로 접근
- **캐시 친화적**: 메모리가 연속적이므로 CPU 캐시 히트율이 높음

## 장점

- **빠른 접근**: 인덱스로 O(1) 시간에 원소에 접근
- **메모리 효율**: 추가 메타데이터가 없어 메모리 오버헤드 최소
- **캐시 효율**: 연속 메모리로 CPU 캐시 활용도 높음
- **캐시 지역성**: 순차 접근 시 성능이 매우 좋음

## 단점

- **고정 크기**: 크기를 미리 정해야 하고 동적 확장 어려움
- **느린 삽입/삭제**: 중간 위치의 삽입/삭제 시 O(n) 비용
- **메모리 낭비**: 필요한 것보다 큰 크기로 할당하면 낭비
- **연속 메모리 필요**: 큰 배열은 충분한 연속 메모리가 필요

## 사용 사례

- **고정된 크기의 데이터**: 월별 판매량, 요일별 데이터 등
- **빠른 인덱스 접근 필요**: LUT(Look-up Table), 캐시 테이블
- **순차 처리**: 이미지 픽셀, 오디오 샘플, 수치 계산
- **정렬된 데이터 검색**: 이진 탐색 기반 알고리즘
- **게임 그래픽**: 정점(Vertex), 픽셀 데이터 저장

## 성능 (Big O)

| 연산 | 시간 복잡도 | 설명 |
|------|-----------|------|
| 접근 (Access) | O(1) | 인덱스로 직접 접근 |
| 검색 (Search) | O(n) | 선형 탐색, 정렬시 O(log n) |
| 삽입 (Insert) | O(n) | 삽입 후 뒤의 원소들 이동 필요 |
| 삭제 (Delete) | O(n) | 삭제 후 뒤의 원소들 이동 필요 |
| 순회 (Traverse) | O(n) | 모든 원소 방문 |
| 공간 (Space) | O(n) | n개 원소 저장 |

## Rust 예시

```rust
// 1. 배열 선언 및 초기화
let arr: [i32; 5] = [1, 2, 3, 4, 5];

// 2. 인덱스로 접근 (O(1))
println!("arr[2] = {}", arr[2]); // 3

// 3. 배열 순회
for (index, &value) in arr.iter().enumerate() {
    println!("arr[{}] = {}", index, value);
}

// 4. 동적 크기 배열 (Vec)
let mut vec = vec![1, 2, 3, 4, 5];
vec.push(6);        // 끝에 추가 O(1) amortized
vec.insert(2, 10);  // 인덱스 2에 삽입 O(n)
vec.remove(2);      // 인덱스 2 제거 O(n)

// 5. 배열 메서드
println!("Length: {}", arr.len());
println!("Sum: {}", arr.iter().sum::<i32>());

// 6. 슬라이싱 (Slice)
let slice = &arr[1..4];  // [2, 3, 4]

// 7. 2차원 배열
let matrix: [[i32; 3]; 2] = [
    [1, 2, 3],
    [4, 5, 6],
];
println!("matrix[0][1] = {}", matrix[0][1]); // 2
```

## 참고

- Rust의 배열과 Vec는 다릅니다
  - 배열: 컴파일 타임 크기 고정
  - Vec: 런타임 동적 크기 조정
- 메모리 캐시 활용으로 인해 이론적 O(1)보다 실제 성능이 더 좋을 수 있음
- 대부분의 알고리즘은 동적 배열(Vec, ArrayList 등)을 사용합니다
