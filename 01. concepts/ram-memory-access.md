# RAM 물리적 구조와 메모리 접근

## 1. RAM의 물리적 구조

RAM(Random Access Memory)은 컴퓨터 시스템의 기본적인 메모리 단위입니다. 물리적으로는 다음과 같이 생각할 수 있습니다:

**RAM은 거대한 1차원 배열**

- 주소로 접근하는 거대한 칸들
- 1바이트씩 저장 가능
- 각 칸은 고유한 주소(address)를 가짐

```
주소:    0    1    2    3    4    5    6    7  ...
내용:  [42] [17] [99] [03] [88] [12] [55] [21] ...
```

예를 들어, 4GB RAM은 약 40억 개의 바이트 칸이 있다는 의미입니다.

## 2. 주소 체계 (Address System)

### Random Access의 의미

**Random Access**: 어떤 주소든 같은 시간에 접근 가능 (약 100나노초)

- 첫 번째 주소든, 마지막 주소든 접근 시간은 동일
- 이것이 HDD와 RAM의 가장 큰 차이
- 주소를 알면 즉시 데이터에 접근 가능

### 주소 계산의 마법: 행/열 좌표 변환

```
주소 변환:
1. CPU가 필요한 주소 계산
2. 주소 신호 → 메모리 컨트롤러로 전송
3. 행(row) 신호 + 열(column) 신호 생성
4. 교차점의 데이터 읽기
5. 데이터 전송 (~100ns)
```

이 과정이 모든 주소에서 동일한 시간이 걸리기 때문에 "Random Access"라고 부릅니다.

## 3. 데이터 구조별 저장 방식

### 배열 (연속 배치)

```c
int arr[5] = {10, 20, 30, 40, 50};
```

**RAM 배치:**

```
주소:  1000  1004  1008  1012  1016
값:    [10]  [20]  [30]  [40]  [50]
```

**접근 방식:**

```
arr[2]에 접근하려면?
= 1000 + (2 × sizeof(int))
= 1000 + (2 × 4)
= 1008번지 → 즉시 접근!
```

**기계어 실행:** 약 3개 명령어
- 배열 시작 주소 로드
- 인덱스 × 크기 계산
- 해당 주소에서 읽기

### 연결 리스트 (흩어진 배치)

```
주소 1000: [값=10, 다음=5000]  ← 첫 노드
주소 5000: [값=20, 다음=2000]  ← 둘째 노드
주소 2000: [값=30, 다음=8000]  ← 셋째 노드
```

**접근 방식:**

세 번째 노드(인덱스 2)에 접근하려면?
1. 첫 노드 주소 1000 접근 → `다음 = 5000` 읽기 (메모리 접근 1)
2. 두 번째 노드 주소 5000 접근 → `다음 = 2000` 읽기 (메모리 접근 2)
3. 세 번째 노드 주소 2000 접근 → `값 = 30` 읽기 (메모리 접근 3)

**기계어 실행:** 6개 이상 명령어
- 포인터를 따라가며 반복적으로 메모리 접근
- 매 스텝마다 메모리 대기 시간 발생

## 4. 포인터 vs 계산

### 배열: 주소 = 시작 + (인덱스 × 크기)

```c
int arr[1000];
int value = arr[500];  // 계산으로 직접 접근
```

- CPU가 수학 계산으로 주소 생성
- 메모리 접근은 1회만 발생

### 연결 리스트: 주소 = 이전 노드가 알려줌

```c
Node* current = head;
for(int i = 0; i < 500; i++) {
    current = current->next;  // 포인터 추적
}
int value = current->data;
```

- 포인터가 다음 노드 주소를 가리킴
- 메모리 접근이 반복적으로 발생
- **Pointer Chasing**: 포인터를 따라가는 비용

## 5. RAM의 물리적 속성

| 특성 | 값 | 의미 |
|------|-----|------|
| **Random Access** | 100ns | 어느 주소든 동일한 접근 시간 |
| **메모리 버스 폭** | 64비트 (8바이트) | 한 번에 8바이트씩 전송 |
| **캐시 라인** | 64바이트 | 메모리 로드의 최소 단위 |
| **메모리 대역폭** | ~100GB/s | 초당 전송량 |

### 연속 배치와 캐시 효율

```
배열 접근:
[데이터1] [데이터2] [데이터3] [데이터4] ...
↓ 캐시 라인 (64바이트)
CPU 캐시에 한 번에 로드!

연결 리스트 접근:
[노드1 @1000] ... [노드2 @5000] ... [노드3 @2000] ...
↓
각각 따로 메모리 접근 필요
```

## 6. 실제 예시: 100만 개 숫자 더하기

### 배열 사용

```c
int arr[1000000];
long sum = 0;
for(int i = 0; i < 1000000; i++) {
    sum += arr[i];
}
```

**메모리 접근:** 약 125,000회
- 버스 폭 8바이트 × int 4바이트 = 2개씩 로드
- 1,000,000 ÷ 8 = 125,000회 접근
- CPU는 계산 진행 중 메모리 prefetch로 데이터 미리 준비

**실행 시간:** ~10-20ms (현대 CPU)

### 연결 리스트 사용

```c
Node* current = head;
long sum = 0;
for(int i = 0; i < 1000000; i++) {
    sum += current->data;
    current = current->next;
}
```

**메모리 접근:** 약 1,000,000회
- 매 노드마다 메모리 접근 필요
- 포인터를 따라가야 함
- CPU가 대기 (prefetch 불가능)

**실행 시간:** ~100-200ms (현대 CPU)

### 성능 차이

```
연결 리스트 / 배열 = 100-200ms / 10-20ms = 5-10배 느림!
```

## 7. 포인터의 비밀

### 포인터와 메모리 접근

```c
int x = 42;        // x는 주소 1000에 저장
int* ptr = &x;     // ptr은 1000을 저장

int value = *ptr;  // 2단계:
                   // 1. ptr에서 1000 읽기
                   // 2. 1000 주소에서 42 읽기
```

### 포인터 체인 (연결 리스트)

```c
struct Node {
    int data;
    Node* next;
};

// 다섯 번째 노드 접근
Node* current = head;
current = current->next;  // 메모리 접근 1
current = current->next;  // 메모리 접근 2
current = current->next;  // 메모리 접근 3
current = current->next;  // 메모리 접근 4
int value = current->data; // 메모리 접근 5
```

**핵심:** 포인터 체인의 길이 = 메모리 접근 횟수

## 8. 접근 방식 시각화

### 배열 접근 흐름

```
CPU 계산
  ↓
주소 = 1000 + (인덱스 × 4)
  ↓
주소 신호 → 메모리 컨트롤러
  ↓
RAM[주소] 직접 접근 (100ns)
  ↓
데이터 반환
```

**총 시간:** ~100ns (메모리 1회 접근)

### 연결 리스트 접근 흐름

```
현재 노드 주소
  ↓
RAM 읽기 → 다음 노드 주소 얻기 (100ns)
  ↓
다음 노드 주소로 이동
  ↓
RAM 읽기 → 또 다음 노드 주소 얻기 (100ns)
  ↓
(반복...)
  ↓
목표 노드 도달 후 데이터 읽기
```

**총 시간:** ~100ns × 접근 깊이 (메모리 N회 접근)

## 9. 핵심 정리 표

| 속성                  | 설명               | 영향                |
| ------------------- | ---------------- | ----------------- |
| **Random Access**   | 어디든 같은 시간(100ns) | 주소만 알면 즉시 접근 가능   |
| **주소 체계**           | 0부터 순차 번호        | 수학 계산으로 위치 결정     |
| **버스 폭**            | 64비트 (8바이트)      | 연속 배치 시 효율적       |
| **캐시 라인**           | 64바이트 단위 로드      | 인접 데이터도 함께 로드     |
| **Pointer Chasing** | 포인터 따라가기         | 메모리 접근 반복 = 성능 저하 |

## 10. 알아두면 좋은 사실들

### CPU 캐시와 메모리 계층

```
레지스터      1ns    (100개 정도)
L1 캐시      5ns    (32KB)
L2 캐시     20ns    (256KB)
L3 캐시     50ns    (8MB)
메인 메모리 100ns   (8GB 이상)
```

연속 배치는 캐시에 들어가기 쉽고, 연결 리스트는 캐시 미스가 자주 발생합니다.

### 메모리 정렬 (Alignment)

```c
struct Bad {
    char a;    // 1바이트
    int b;     // 4바이트 (3바이트 패딩)
    char c;    // 1바이트 (3바이트 패딩)
};

struct Good {
    int b;     // 4바이트
    char a;    // 1바이트 (1바이트 패딩)
    char c;    // 1바이트 (2바이트 패딩)
};
```

데이터를 버스 폭에 맞춰 정렬하면 메모리 접근이 더 빠릅니다.
