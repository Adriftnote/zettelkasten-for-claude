# 컴퓨터 메모리 계층 구조

**속도와 용량의 트레이드오프로 설계된 계층적 저장 시스템**

## 📖 개요

컴퓨터는 서로 다른 속도와 용량을 가진 여러 종류의 메모리를 계층적으로 사용합니다. 빠를수록 용량이 작고 비싸며, 느릴수록 용량이 크고 저렴합니다.

## 🏗️ 메모리 계층 구조

```
빠름 ⚡                         느림 🐌
작음                            큼
비쌈                            저렴
휘발성                          비휘발성
    ↓                          ↓

CPU 레지스터 (8-32개, 0.5ns)
    ↓
L1 캐시 (32KB, 1ns)
    ↓
L2 캐시 (256KB, 2-4ns)
    ↓
L3 캐시 (8MB, 10-20ns)
    ↓
RAM (16GB, 100ns)
    ↓
SSD (1TB, 100,000ns)
    ↓
HDD (4TB, 10,000,000ns)
```

## 💾 각 계층 상세

### 1. CPU 레지스터

```
위치: CPU 칩 내부
타입: 극초고속 메모리
용량: 8-32개 (각 8바이트)
속도: 0.5ns
```

**역할**: CPU가 즉시 사용하는 데이터/명령어 저장

**비유**: 손에 든 것 (지금 이 순간 작업 중인 것)

### 2. CPU 캐시 (L1/L2/L3)

```
위치: CPU 칩 내부 (RAM과 별개!)
타입: SRAM (Static RAM)
구조:
├─ L1: 32KB, 1ns (코어별)
├─ L2: 256KB, 2-4ns (코어별)
└─ L3: 8MB, 10-20ns (공유)
```

**역할**: RAM에서 자주 쓰는 데이터를 임시 복사

**특징**:
- 64바이트 단위로 캐시 라인 로드
- 연속된 메모리 접근 시 효율적 (공간 지역성)
- CPU가 자동으로 관리

**비유**: 책상 위 (지금 펼쳐놓은 책들)

### 3. RAM (메인 메모리)

```
위치: 메인보드에 별도 장착
타입: DRAM (Dynamic RAM)
용량: 8-128GB (일반 PC)
속도: 100ns
휘발성: 전원 꺼지면 사라짐
```

**역할**: 실행 중인 프로그램과 데이터 저장

**특징**:
- 프로그램 실행 시 SSD에서 로드됨
- 데이터 구조가 주로 여기 존재
- 전원 꺼지면 모든 내용 사라짐

**비유**: 책상 전체 (작업 공간)

### 4. SSD (Solid State Drive)

```
타입: 플래시 메모리 (NAND)
용량: 256GB - 4TB
속도: 100,000ns (0.1ms)
비휘발성: 전원 꺼져도 유지
```

**역할**: 운영체제, 프로그램, 파일 영구 저장

**특징**:
- 기계적 부품 없음 (조용함)
- HDD보다 100배 빠름
- 쓰기 횟수 제한 있음

**비유**: 책장 (빠르게 꺼낼 수 있는 책들)

### 5. HDD (Hard Disk Drive)

```
타입: 자기 디스크 (기계식)
용량: 1TB - 20TB
속도: 10,000,000ns (10ms)
비휘발성: 전원 꺼져도 유지
```

**역할**: 대용량 데이터 영구 저장 (백업, 아카이브)

**특징**:
- 회전하는 디스크 + 헤드
- 저렴하지만 느림
- 기계적 고장 위험

**비유**: 창고 (오래 보관하는 물건들)

## 🔄 데이터 흐름

### 프로그램 실행 시

```
1. SSD에 저장된 프로그램 파일
   └→ 2. RAM으로 로드 (프로그램 시작)
        └→ 3. 데이터 구조 생성 (RAM)
             └→ 4. CPU가 필요한 부분을 캐시로 복사
                  └→ 5. 레지스터로 이동
                       └→ 6. CPU 연산
                            └→ 7. 결과를 다시 RAM에 저장
                                 └→ 8. 필요시 SSD에 저장
```

### 파일 저장 시

```
1. 사용자: "저장" 클릭
   ↓
2. RAM의 데이터
   ↓
3. SSD로 쓰기 (영구 저장)
   ↓
4. 프로그램 종료해도 파일 유지 ✅
```

## 🧠 인간 기억 시스템과의 비교

| 컴퓨터 | 인간 기억 | 특징 |
|--------|----------|------|
| **레지스터** | 의식 (Consciousness) | 지금 이 순간 |
| **캐시** | 주의/초점 (Attention) | 집중하고 있는 것 |
| **RAM** | 작업 기억 (Working Memory) | 7±2 항목 제한 |
| **SSD** | 장기 기억 - 최근 (LTM) | 빠른 회상 |
| **HDD** | 장기 기억 - 오래된 (LTM) | 느린 회상 |

### 시나리오 비교

**사람이 책 읽을 때:**
```
1. 책장에서 책 꺼내기 (SSD → RAM)
2. 책상에 펼치기 (RAM)
3. 한 페이지 읽기 (캐시)
4. 지금 보는 단어 (레지스터)
5. 메모 작성 (RAM → SSD 저장)
```

**컴퓨터가 파일 열 때:**
```
1. SSD에서 파일 읽기
2. RAM에 로드
3. 현재 보는 부분을 캐시에 복사
4. CPU가 처리
5. 수정 사항을 RAM에 반영
6. 저장 시 SSD에 쓰기
```

## ⚡ 캐시의 동작 원리

### 캐시 라인 (Cache Line)

```
CPU가 1000번지를 요청하면...

실제로 로드되는 것:
[1000][1001][1002]...[1063] (64바이트)

다음에 1001~1063 접근 시:
→ 캐시에 있음! 초고속! ⚡
```

### 공간 지역성 (Spatial Locality)

```
"근처 데이터는 곧 쓸 가능성이 높다"

배열: [1][2][3][4][5] 연속 배치
→ 1 읽으면 2-5도 캐시에 로드
→ 캐시 히트율 높음 ⚡

연결 리스트: [1]→[5000]→[2000]...
→ 매번 다른 위치 접근
→ 캐시 미스 많음 😰
```

### 시간 지역성 (Temporal Locality)

```
"방금 쓴 데이터는 곧 다시 쓸 가능성이 높다"

반복문:
for i in 0..1000 {
    sum += arr[i];  // arr[i] 반복 접근
}

→ 캐시에 유지되어 빠름
```

## 📊 실제 성능 차이

### 1000만 개 숫자 더하기 (실험)

| 저장 위치 | 시간 | 이유 |
|----------|------|------|
| **캐시** | 10ms | 99% 캐시 히트 |
| **RAM** | 50ms | 일부 캐시 미스 |
| **SSD** | 5,000ms | 디스크 I/O |
| **HDD** | 50,000ms | 기계적 지연 |

→ **5000배 차이!**

### 캐시 친화적 vs 비친화적 코드

```rust
// ✅ 캐시 친화적 (10ms)
let arr = vec![1, 2, 3, ..., 10_000_000];
for num in &arr {
    sum += num;  // 순차 접근
}

// ❌ 캐시 비친화적 (200ms)
let list = LinkedList::from([1, 2, ..., 10_000_000]);
for num in &list {
    sum += num;  // 랜덤 접근
}
```

## 🎯 최적화 전략

### 1. 데이터 구조 선택

```
빠른 순차 접근 필요 → Vec/배열 ✅
자주 삽입/삭제 → VecDeque
키-값 검색 → HashMap
```

### 2. 메모리 배치

```
// ✅ 좋은 예: 구조체 배열
struct Point { x: f32, y: f32 }
let points = vec![Point { x: 1.0, y: 2.0 }, ...];

// ❌ 나쁜 예: 포인터 배열
let points = vec![Box::new(Point {...}), ...];
```

### 3. 루프 최적화

```rust
// ✅ 캐시 친화적
for i in 0..1000 {
    for j in 0..1000 {
        matrix[i][j] += 1;  // 행 우선
    }
}

// ❌ 캐시 비친화적
for j in 0..1000 {
    for i in 0..1000 {
        matrix[i][j] += 1;  // 열 우선 (점프)
    }
}
```
