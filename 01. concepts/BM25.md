# BM25

텍스트 검색을 위한 **랭킹 알고리즘**으로, [[TF-IDF]]의 개선판입니다.

## 📖 개요

BM25 (Best Matching 25)는 검색 엔진에서 문서의 관련성을 점수화하는 확률적 랭킹 함수입니다. 쿼리의 각 단어가 문서에 얼마나 잘 매칭되는지를 계산합니다.

## 🎭 비유

도서관에서 "고양이 사료"를 검색하면:
- BM25는 **정확히 "고양이"와 "사료"라는 글자**가 있는 책을 찾습니다
- 자주 등장하지만 너무 흔하지 않은 단어에 높은 점수를 줍니다

## ⚖️ 벡터 검색과 비교

| 특성           | BM25         | 벡터 검색           |
| ------------ | ------------ | --------------- |
| **방식**       | 키워드 매칭       | 의미 유사도          |
| **"고양이" 검색** | "고양이" 글자 필요  | "냥이", "cat"도 매칭 |
| **장점**       | 정확한 용어 검색    | 의미적 유사성 포착      |
| **단점**       | 동의어 놓침       | 계산 비용 높음        |
| **적합한 경우**   | 기술 문서, 코드 검색 | 자연어 질문          |

## ✨ 특징

- **TF (Term Frequency)**: 단어가 문서에 많이 나올수록 점수 ↑
- **IDF (Inverse Document Frequency)**: 희귀한 단어일수록 점수 ↑
- **문서 길이 정규화**: 긴 문서에 불이익 방지
- **포화 함수**: 단어 빈도가 일정 수준 이상이면 점수 증가 둔화

## 💡 사용 사례

```
검색 쿼리: "Python async await"

BM25 결과:
1. "Python에서 async/await 사용법" (정확 매칭)
2. "비동기 프로그래밍 with Python" (부분 매칭)

벡터 검색 결과:
1. "JavaScript의 Promise 패턴" (의미 유사)
2. "Python 코루틴 가이드" (의미 유사)
```
