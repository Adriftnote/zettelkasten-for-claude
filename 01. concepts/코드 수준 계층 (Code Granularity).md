# 코드 수준 계층 (Code Granularity)

소프트웨어 코드를 규모와 복잡도에 따라 구분하는 계층 구조

## 📖 개요

코드는 규모에 따라 함수, 파일, 레포지토리 수준으로 구분됩니다. 각 수준은 관리 방식, 복잡도, 의존성 처리 방법이 다릅니다. 레포지토리 수준은 완전한 소프트웨어 프로젝트를 의미하며, 수십~수백 개의 파일이 서로 연결되어 작동합니다.

## 🎭 비유

요리에 비유하면:
- **함수**: 레시피 1개 (피자 만들기) - 3~10줄
- **파일**: 요리책 1권 (이탈리아 요리) - 100~500줄
- **레포지토리**: 레스토랑 운영 전체 매뉴얼 - 10,000~100,000줄

## ✨ 특징

### 함수 수준 🔧
- 3~10줄의 작은 코드
- 단일 작업 수행
- 독립적으로 테스트 가능

### 파일 수준 📄 (≈ 모듈)
- 100~500줄
- 관련 함수들의 집합
- 하나의 책임 영역 담당

### 레포지토리 수준 🏢
- 10,000~100,000줄 이상
- 완전한 소프트웨어 프로젝트
- 수십~수백 개 파일이 연결
- 예: 카카오톡, 넷플릭스

## 💡 예시

```
레포지토리 수준:
쇼핑몰_앱/ (10,000~100,000줄)
├── 사용자관리/ (50개 파일)
│   └── login.py ← 파일 수준
│       └── login() ← 함수 수준
├── 상품관리/
├── 주문결제/
└── 데이터베이스/
```
